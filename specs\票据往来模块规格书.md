## 1. 模块概述

### 1.1 业务定位

票据往来模块是ERP财务管理中**非现金收款**的核心枢纽，负责管理商业汇票、支票、本票等票据从收取到最终处置的全生命周期。

```
┌──────────────────────────────────────────────────────────┐
│                    票据模块架构定位                         │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  上游: 实收收款模块 ──── 收到票据 ────→ 票据往来模块       │
│                                         │                │
│                              ┌──────────┼──────────┐     │
│                              ↓          ↓          ↓     │
│                           背书转让    贴现/质押   到期兑付 │
│                              │          │          │     │
│  下游:                      ↓          ↓          ↓     │
│  ┌──────────────┐   ┌──────────┐  ┌──────────┐         │
│  │ 采购付款模块  │   │ 银企对接  │  │ 总账凭证  │         │
│  │ (应付核销)   │   │ (资金)   │  │ (入账)   │         │
│  └──────────────┘   └──────────┘  └──────────┘         │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 1.2 核心目标

| # | 目标 | 衡量指标 |
|:--:|:---|:---|
| 1 | 票据全生命周期可视化管理 | 100%票据状态实时可查 |
| 2 | 到期零遗漏 | 票据到期前分级预警，逾期率=0 |
| 3 | 流转过程留痕 | 背书/贴现/质押全程可追溯 |

### 4.4 票面要素校验规则

```python
def validate_bill_note(bill: dict) -> list[str]:
    """票面要素完整性校验，返回错误列表"""
    errors = []
    
    # 必填字段
    required = ['bill_type', 'bill_number', 'draft_date', 'due_date', 
                'face_amount', 'drawer', 'acceptor']
    for field in required:
        if not bill.get(field):
            errors.append(f"缺少必填字段: {field}")
    
    # 业务规则校验
    if bill.get('face_amount', 0) <= 0:
        errors.append("票面金额必须大于0")
    
    if bill.get('due_date') and bill.get('draft_date'):
        if bill['due_date'] <= bill['draft_date']:
            errors.append("到期日期必须晚于出票日期")
        
        days = (bill['due_date'] - bill['draft_date']).days
        if days > 365:
            errors.append("票据期限不得超过一年")
    
    # 银行承兑汇票必须填写承兑银行
    if bill.get('bill_type') == 1 and not bill.get('acceptor_bank'):
        errors.append("银行承兑汇票必须填写承兑银行")
    
    # 金额一致性（OCR场景）
    if bill.get('amount_chinese') and bill.get('face_amount'):
        # 大写金额与小写金额比对
        if not amount_match(bill['amount_chinese'], bill['face_amount']):
            errors.append("大写金额与小写金额不一致")
    
    return errors
```

### 4.5 贴现计算规则

```python
def calculate_discount(
    face_amount: float,     # 票面金额
    discount_rate: float,   # 贴现年利率 (例: 0.032 = 3.2%)
    discount_date: date,    # 贴现日期
    due_date: date          # 到期日期
) -> dict:
    """
    贴现利息计算
    公式: 贴现利息 = 票面金额 × 贴现利率 × 贴现天数 / 360
    """
    discount_days = (due_date - discount_date).days
    discount_interest = round(face_amount * discount_rate * discount_days / 360, 2)
    actual_amount = round(face_amount - discount_interest, 2)
    
    return {
        'face_amount': face_amount,
        'discount_rate': discount_rate,
        'discount_days': discount_days,
        'discount_interest': discount_interest,
        'actual_amount': actual_amount,
    }
```

## 5. 凭证模板矩阵

### 5.1 六种标准凭证模板

| 编号 | 场景 | 借方 | 贷方 | 凭证字 |
|:---:|:---|:---|:---|:---:|
| **B01** | 票据收取 | 应收票据-银承/商承（票面金额） | 应收账款-XX客户（核销金额） | 收 |

## 7. 到期预警机制

### 7.1 分级预警规则

| 级别 | 提前天数 | 通知渠道 | 通知对象 | 频率 |
|:---:|:---:|:---|:---|:---:|
| 🟢 L1 | 30天 | 站内信 | 出纳 | 每日1次 |
| 🟡 L2 | 15天 | 站内信 + 邮件 | 出纳 + 财务主管 | 每日1次 |
| 🟠 L3 | 7天 | 站内信 + 邮件 + 短信 | 出纳 + 财务主管 | 每日1次 |
| 🔴 L4 | 3天 | 全渠道 + 电话 | 出纳 + 财务主管 + 部门负责人 | 每日2次 |

### 7.2 定时任务设计

```python
from apscheduler.schedulers.background import BackgroundScheduler

scheduler = BackgroundScheduler()

@scheduler.scheduled_job('cron', hour=8, minute=0, id='bill_alert_daily')
def daily_bill_alert():
    """每日8:00执行票据到期预警扫描"""
    today = date.today()
    
    # 查询所有持有中+质押中的票据
    bills = BillNote.query.filter(
        BillNote.status.in_([BillStatus.HOLDING, BillStatus.PLEDGED]),
        BillNote.is_deleted == 0
    ).all()
    
    alert_configs = BillAlertConfig.query.filter_by(is_enabled=1).all()
    
    for bill in bills:
        days_remaining = (bill.due_date - today).days
        
        for config in alert_configs:
            if days_remaining <= config.alert_days:
                # 检查是否已发送过该级别预警
                if not has_alert_sent(bill.id, config.alert_level):
                    send_alert(bill, config)
                    record_alert(bill.id, config.alert_level)
                break  # 只触发最高级别
        
        # 逾期检查
        if days_remaining < 0 and bill.status == BillStatus.HOLDING:
            escalate_overdue(bill)
```

## 8. 安全与风控

### 8.1 票据实物安全

| 措施 | 说明 |
|:---|:---|
| 保管位置登记 | 纸质票据必须记录保险柜编号 |
| 保管人责任制 | 每张票据指定保管人，交接需签收 |
| 盘点机制 | 月度票据实物盘点，系统核对 |
| 遗失处理 | 48小时内挂失止付 + 公示催告 |

### 8.2 信用风险控制

| 风控点 | 规则 |
|:---|:---|
| 商业承兑汇票限额 | 单一承兑人持有票据总额不超过应收账款50% |
| 承兑人信用评级 | 接入企业征信，B级以下需审批 |
| 大额票据审批 | 票面金额≥50万需财务总监审批 |
| 异常到期率 | 同一承兑人连续2次拒付，暂停接受其票据 |

### 8.3 操作权限

| 操作 | 角色 | 说明 |
|:---|:---|:---|
| 票据登记 | 出纳/财务 | 基础录入权限 |
| 票据背书 | 出纳（申请）+ 财务主管（审批） | 双人控制 |
| 票据贴现 | 出纳（申请）+ 财务主管（审批） | 大额需总监 |
| 票据质押 | 财务主管（申请）+ 财务总监（审批） | 高风险操作 |
| 到期托收 | 出纳 | 自动提醒后执行 |
| 凭证修改 | 仅财务主管 | 生效凭证不可删 |
| 台账查询 | 出纳/财务/主管 | 按角色过滤数据 |
