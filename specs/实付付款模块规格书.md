## 1. 模块定位与业务目标

### 1.1 一句话定义

**实付付款模块是ERP系统中"资金流出"的核心枢纽，负责从采购应付款项出发，经过付款申请、多级审批、资金执行、应付核销、凭证生成，最终写入总账的全流程管理。**

### 1.2 在整体流程中的位置

```
采购申请 → 采购订单 → 收货验收 → 应付对账 ─┐
                                          │
进项发票 → 发票录入/查验 ────────────────┤
                                          ▼
                                  ┌──────────────┐
                                  │ 💳 实付付款   │ ◄── 本模块
                                  │ (本规格书)    │
                                  └──────┬───────┘
                                         │
                                         ▼
                                  ┌──────────────┐
                                  │ 🧾 总账凭证    │ ← P0待建
                                  │              │
                                  │ 借:应付账款   │
                                  │ 贷:银行存款   │
                                  └──────────────┘
```

### 1.3 与实收收款的镜像关系

| 维度 | 实收收款（资金流入） | 实付付款（资金流出） |
|:---|:---|:---|
| 方向 | 钱进来 → 分配给应收 | 应付确认 → 申请审批 → 钱出去 |
| 核心动作 | 核销（分钱） | 核销（分单） |
| 审批需求 | 低（已到账的钱） | **高（未出的钱需审批）** |
| 资金安全 | 被动接收，风险低 | **主动划出，风险高** |
| 预收/预付 | 客户多付 → 预收 | 多付给供应商 → 预付 |
| 票据 | 收到承兑汇票（应收票据） | 付出承兑汇票（应付票据） |

### 1.4 核心业务目标

| 目标 | 指标 | 当前痛点 |
|:---|:---|:---|
| **付款可控** | 100%付款经过审批流，无越权支付 | 口头通知出纳付款，无留痕 |
| **资金安全** | 付款信息（账户/金额/收款方）三重校验 | 手工填网银，容易输错 |
| **核销准确** | 应付余额与实际付款实时同步 | 付了款但应付余额没更新 |
| **预付透明** | 预付余额实时可见，到期自动提醒 | 预付了多少钱不知道，容易遗忘 |
| **凭证自动化** | 付款确认后自动生成凭证 | 手工做分录 |
| **供应商账龄** | 准确的供应商应付账龄分析 | 月末才统计，平时不准 |

### 4.3 审批阈值规则

```python
class ApprovalThreshold:
    """付款审批阈值配置"""
    LEVELS = [
        (5000,      "部门经理",   2),     # ≤5千, 部门经理, 2小时内
        (50000,     "财务主管",   4),     # ≤5万, 财务主管, 4小时内
        (200000,    "总经理",     8),     # ≤20万, 总经理, 8小时内(1工作日)
        (float('inf'), "董事会",  16),    # >20万, 董事会, 16小时内(2工作日)
    ]
    
    @staticmethod
    def get_level(amount: float) -> tuple:
        """根据金额返回(角色, 超时小时数)"""
        for threshold, role, hours in ApprovalThreshold.LEVELS:
            if amount <= threshold:
                return role, hours
        return "董事会", 16
```

### 4.4 核销规则引擎

```python
class PayWriteoffEngine:
    """付款核销规则引擎"""
    
    # 匹配权重
    WEIGHT_SUPPLIER = 0.40  # 供应商匹配 40%
    WEIGHT_AMOUNT = 0.30    # 金额匹配 30%
    WEIGHT_SUMMARY = 0.15   # 摘要匹配 15%
    WEIGHT_HISTORY = 0.15   # 历史核销习惯 15%
    
    # 尾差阈值
    TOLERANCE_AMOUNT = 1.00  # ≤1元尾差自动处理
    
    # 置信度等级
    CONFIDENCE_AUTO = 90     # ≥90%: 自动推荐
    CONFIDENCE_CONFIRM = 70  # ≥70%: 提示确认
    # <70%: 手动选择
    
    @staticmethod
    def calculate_confidence(payment, ap_bill) -> float:
        """计算核销匹配置信度"""
        score = 0.0
        # 供应商必须一致
        if payment.supplier_id == ap_bill.supplier_id:
            score += PayWriteoffEngine.WEIGHT_SUPPLIER * 100
        else:
            return 0.0  # 供应商不同直接排除
        
        # 金额接近度
        if payment.total_amount > 0:
            amount_ratio = min(ap_bill.balance / payment.total_amount, 1.0)
            score += PayWriteoffEngine.WEIGHT_AMOUNT * amount_ratio * 100
        
        # 摘要关键词匹配 (简化)
        # ...
        
        return round(score, 2)
```

## 8. 安全与风控

### 8.1 资金安全规则

| 规则 | 说明 | 实现方式 |
|:---|:---|:---|
| **三岗分离** | 制单人、审批人、付款人必须不同 | 角色权限校验 |
| **大额双签** | ≥10万需财务主管+总经理双签 | 审批流自动配置 |
| **收款方校验** | 首次付款需供应商确认银行信息 | 供应商档案冻结前3次 |
| **重复检测** | 同供应商同金额同日≤1笔自动检测 | 实时校验+预警 |
| **操作留痕** | 所有操作完整记录（操作人+时间+IP） | 审计日志表 |
| **数据加密** | 银行账号等敏感信息加密存储 | AES-256 |

### 8.2 预警规则

| 预警 | 条件 | 通知方式 |
|:---|:---|:---|
| 审批超时 | 超过阈值时长未审批 | 系统消息+邮件/企业微信 |
| 预付超期 | 预付超过90天未抵扣 | 每周一推送到财务主管 |
| 应付逾期 | 应付单超过约定付款日 | 到期前3天+到期日+逾期每天 |
| 银行失败 | 付款指令银行返回失败 | 实时推送到出纳 |
| 重复风险 | 检测到疑似重复付款 | 弹窗确认+需主管二次确认 |
