## 1. 模块定位与业务目标

### 1.1 一句话定义

**费用报销模块是ERP系统中"申请-审批-支付-记账"闭环的费用管控中枢，负责从员工费用申请开始，经过多级审批、票据关联、预算扣减、费用分摊，最终生成报销付款和费用凭证写入总账的全流程管理。**

### 1.3 与已有模块的衔接关系

| 上游模块 | 衔接点 | 本模块消费的数据 |
|:---|:---|:---|
| **发票查验模块** | 报销关联发票验真 | 查验结果、票面信息、发票状态 |
| **预算管理模块** | 费用申请时预算校验 | 预算科目、预算余额、预算控制策略 |
| **组织架构/HR** | 审批流+员工信息 | 部门层级、审批权限、差旅标准 |
| **成本中心** | 费用分摊维度 | 成本中心编码、分摊比例 |

| 下游模块 | 衔接点 | 本模块产出的数据 |
|:---|:---|:---|
| **实付付款模块** | 报销审批通过 → 付款申请 | 报销单号、金额、收款人、银行账户 |
| **总账凭证引擎** | 报销确认 → 自动凭证 | EX01~EX07 凭证模板触发事件 |
| **预算管理模块** | 报销完成 → 预算扣减回写 | 占用金额、实际发生金额 |
| **成本核算模块** | 费用归集 → 成本分摊 | 费用科目、成本中心、分摊规则 |

### 1.4 核心业务目标

| 目标 | 指标 | 当前痛点 |
|:---|:---|:---|
| **报销时效** | 从申请到到账不超过5个工作日 | 纸质单据流转慢，审批堵点不可见 |
| **预算准确率** | 费用100%受预算控制，超预算0%漏过 | 报销与预算脱节，事后才知道超支 |
| **发票合规率** | 关联发票100%通过查验 | 手工审核发票，重复报销/假票难发现 |
| **审批效率** | 平均审批链路不超过3级 | 审批层级过多，领导出差即卡住 |
| **凭证自动化** | 报销凭证100%自动生成 | 手工录入费用凭证，科目容易选错 |

## 4. 凭证模板体系

### EX01 — 日常费用报销凭证

| 触发事件 | 报销单确认支付 |
|:---|:---|
| **借方** | |
| 6602 管理费用-xx费 | 报销费用明细(按费用类型分列) |
| 2221-01 应交税费-进项税 | 发票可抵扣进项税额 |
| **贷方** | |
| 1002 银行存款 | 实付金额(actual_pay) |

> 示例：员工张三报销办公用品500元(含税)，发票为专票税额57.52元
> - 借：管理费用-办公费 442.48
> - 借：应交税费-进项税 57.52
> - 贷：银行存款 500.00

## 6. 关键业务规则

### 6.1 审批流规则

| 条件 | 审批链路 | 说明 |
|:---|:---|:---|
| 日常费用 ≤ 2,000元 | 部门主管 → 财务审核 | 2级 |
| 日常费用 2,000~10,000元 | 部门主管 → 部门经理 → 财务审核 | 3级 |
| 日常费用 > 10,000元 | 部门主管 → 部门经理 → 分管副总 → 财务审核 | 4级 |
| 差旅费用 | 部门主管 → 部门经理 → 财务审核 | 3级 |
| 招待费 | 部门主管 → 部门经理 → 分管副总 → 财务审核 | 4级 |
| 借款 | 部门主管 → 财务审核 → 出纳确认 | 3级 |

### 6.2 差旅标准（示例）

| 职级 | 交通标准 | 住宿标准(一线) | 住宿标准(二线) | 餐补/天 |
|:---|:---|:---|:---|:---|
| 普通员工 | 火车硬座/二等座 | 300元/天 | 200元/天 | 80元 |
| 主管 | 火车一等座 | 400元/天 | 300元/天 | 100元 |
| 经理 | 高铁一等座 | 500元/天 | 400元/天 | 120元 |
| 总监 | 飞机经济舱 | 700元/天 | 500元/天 | 150元 |
| 高管 | 飞机商务舱 | 实报实销 | 实报实销 | 200元 |

### 6.3 预算控制策略

| 策略 | 说明 | 适用场景 |
|:---|:---|:---|
| **禁止** | 超预算直接拦截，不允许提交 | 严格控制类(招待费/福利费) |
| **警告** | 超预算提示警告，可继续提交但需说明 | 一般控制类(办公费/差旅费) |
| **特批** | 超预算需额外审批层级 | 预算外特殊费用 |
| **不控制** | 不做预算校验 | 不可预见费用 |

### 6.4 发票处理规则

| 规则 | 说明 |
|:---|:---|
| 专票可抵扣 | 增值税专用发票的进项税额可抵扣，税额单独列示 |
| 普票不可抵扣 | 增值税普通发票全额计入费用 |
| 机票可抵扣 | 国内旅客运输机票可按9%计算抵扣 |
| 招待费不可抵扣 | 业务招待费即使取得专票也不可抵扣 |
| 重复发票拦截 | 同一发票代码+号码不可关联两次报销 |
| 发票金额核对 | 发票金额合计应不超过报销金额 |

## 7. 状态机设计

### 7.1 费用申请状态流转

```
草稿 ──提交──▶ 审批中 ──全部通过──▶ 已通过 ──生成报销──▶ 已报销
  │              │                                    │
  │              ├──驳回──▶ 已驳回 ──修改重提──▶ 审批中   │
  │              │                                    │
  └──撤销──▶ 已撤销                          ──撤销──▶ 已撤销
```

### 7.2 报销单状态流转

```
待支付 ──提交付款──▶ 支付中 ──银行回执──▶ 已支付
  │                    │
  │                    └──退回──▶ 已退回
  └──撤销──▶ 已撤销
```

### 7.3 预算占用生命周期

```
                  审批通过                报销确认
预算余额 ────────▶ 预算占用 ────────▶ 预算扣减
                  (冻结额度)           (实际发生)
                     │
                     └──驳回/撤销──▶ 预算释放(归还余额)
```

## 8. 非功能性需求

| 类别 | 要求 |
|:---|:---|
| **性能** | 费用申请列表查询不超过2秒，预算校验实时响应不超过500ms |
| **并发** | 支持500人同时提交报销申请，预算余额校验需加行锁防超扣 |
| **审计** | 所有审批操作、预算占用/释放/扣减全程留痕，不可删除 |
| **安全** | 员工只能查看自己的申请，审批人只能操作自己节点的审批 |
| **可用性** | 移动端可提交申请和审批，PC端做复杂报表和配置 |
| **集成** | 对接发票查验模块(已建)、预算管理模块(已建)、实付付款模块(已建) |
