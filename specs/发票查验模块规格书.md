# 发票查验模块 - 技术规格说明书

## Invoice Verification Module — Technical Specification

> **项目**: Dev_Dept ERP 财务管理系统  
> **模块**: 发票查验（Invoice Verification）  
> **版本**: v1.0  
> **日期**: 2026-04-14  
> **对标系统**: 金蝶云星空 → 发票管理 → 发票采集/查验  
> **状态**: P0优先级 — 合规底线需求  

---

## 1. 模块概述

### 1.1 业务定位

发票查验模块是ERP财务系统的**合规防火墙**，负责对进入系统的所有增值税发票进行真伪核验、状态检查和信息提取。它位于"发票录入"节点之后、"应付/应收挂账"之前，是业财数据质量的第一道关卡。

```
┌─────────────┐     ┌──────────────┐     ┌──────────────┐
│   OCR/手工    │────▶│  🛡️ 发票查验   │────▶│  应收/应付    │
│   录入发票    │     │  (本模块)      │     │  挂账入账     │
└─────────────┘     └──────────────┘     └──────────────┘
                          │
                   ┌──────┴──────┐
                   ▼             ▼
              ┌─────────┐  ┌──────────┐
              │ 真伪校验 │  │ 状态检测  │
              └─────────┘  │ 作废/红冲 │
                           └──────────┘
```

### 1.2 核心能力

| 能力 | 描述 | 优先级 |
|------|------|--------|
| 🔍 发票真伪查验 | 通过税局数据库实时核验发票是否存在、信息是否匹配 | P0 |
| ⚠️ 异常状态检测 | 自动识别作废、红冲、异常等状态发票 | P0 |
| 📄 全票种支持 | 支持专票、普票、电子票、机动车、通行费等12种发票类型 | P0 |
| 🔄 批量查验 | 支持Excel/图片批量导入，异步批量处理 | P1 |
| 📊 查验台账 | 完整的查验记录留存，满足审计追溯要求 | P0 |
| 🔗 与业务单据联动 | 查验结果自动回写采购入库单/销售发货单 | P1 |

## 2. 接口选型

### 2.1 选型结论：腾讯云 VatInvoiceVerifyNew（新版）

> 你之前访问的老接口 `VatInvoiceVerify` 已下线，**新版接口为 `VatInvoiceVerifyNew`**。

| 维度 | 说明 |
|------|------|
| **服务商** | 腾讯云 OCR 文字识别 |
| **接口名称** | `VatInvoiceVerifyNew` |
| **请求域名** | `ocr.tencentcloudapi.com` |
| **API版本** | `2018-11-19` |
| **QPS限制** | 20次/秒（默认） |
| **SDK支持** | Python / Java / Node.js / Go / PHP / .NET / C++ / Ruby |
| **计费方式** | 按调用次数计费（需在控制台开通） |

### 2.2 选型理由

- ✅ 我们技术栈与腾讯生态天然亲近（公众号、小程序均在腾讯平台）
- ✅ 支持12种发票类型，覆盖全电发票（数电票）
- ✅ 返回完整票面信息，不仅验真还能提取结构化数据
- ✅ Python SDK 成熟，对接效率高
- ✅ 相比阿里云/百度的同类接口，文档更清晰

### 2.3 备选方案

| 方案 | 适用场景 | 切换条件 |
|------|---------|----------|
| 阿里云 `VerifyVATInvoice` | 多云容灾 | 腾讯云服务不可用时 |
| 电子税务局直连API | 零成本、最高权威性 | 日均查验量 > 500次后考虑切换 |
| 国税局查验平台 + RPA | 零成本但自动化程度低 | MVP阶段临时方案 |

## 3. 技术规格

### 3.1 支持的发票类型（12种）

| 票种代码 | 发票类型 | 必填特殊字段 | 备注 |
|:---|:---|:---|:---|
| `01` | 增值税专用发票 | Amount(不含税金额) | 最常用 |
| `02` | 货物运输业增值税专用发票 | Amount | 较少使用 |
| `03` | 机动车销售统一发票 | Amount | 含车辆信息返回 |
| `04` | 增值税普通发票 | CheckCode(后6位) | 个人消费常见 |
| `08` | 增值税电子专用发票（含全电） | Amount(全电时为价税合计) | **全面推广中★** |
| `10` | 增值税电子普通发票（含全电） | CheckCode(后6位) | **全面推广中★** |
| `11` | 增值税普通发票（卷式） | CheckCode(后6位) | 餐饮/零售常见 |
| `14` | 增值税电子（通行费）发票 | CheckCode | ETC相关 |
| `15` | 二手车销售统一发票 | Amount | 含车辆信息返回 |
| `16` | 财政发票 | Amount | 医疗票据等 |
| `32` | 区块链发票（深圳） | SellerTaxCode + Amount | 注：云南已下线 |
| — | **通用机打电子发票** | RegionCode + SellerTaxCode | 地方性电子发票 |

#### 请求地址


#### 公共参数


#### 业务请求参数

| 参数名 | 类型 | 必填 | 说明 | 示例值 |
|:---|:---|:---:|:---|:---|
| `InvoiceNo` | String | ✅ | 发票号码（8位或20位全电票） | `"04138864"` |
| `InvoiceDate` | String | ✅ | 开票日期，格式 YYYY-MM-DD，不支持当天 | `"2025-01-15"` |
| `InvoiceCode` | String | 条件 | 发票代码（10或12位），**全电发票为空** | `"144002288010"` |
| `InvoiceKind` | String | 否 | 票种类型代码 | `"04"` |
| `CheckCode` | String | 条件 | 校验码后6位（普票/卷票/通行费必填） | `"921630"` |
| `Amount` | String | 条件 | 不含税金额（专票必填）；全电票填**价税合计** | `"498.90"` |
| `RegionCode` | String | 条件 | 地区编码（通用机打电子发票必填） | `"4400"` |
| `SellerTaxCode` | String | 条件 | 销方税号（区块链/通用机打必填） | `"44078***401X"` |
| `EnableCommonElectronic` | Boolean | 否 | 是否开启通用机打电子发票，默认false | `true` |
| `EnableTodayInvoice` | Boolean | 否 | 是否允许查验当日发票，默认false | `true` |

### 3.4 关键限制条件

| 限制项 | 规则 | 处理建议 |
|:---|:---|:---|
| **时间范围** | 仅支持**5年内**开具的发票 | 超范围提示用户走人工渠道 |
| **当日发票** | 默认**不支持**查验当天开票 | 设置 `EnableTodayInvoice=true` 可开启 |
| **查验频次** | 同一发票每日最多**5次** | 系统内做去重缓存，重复查询直接返回缓存结果 |
| **格式要求** | 日期必须 `YYYY-MM-DD`，金额精确到分 | 前端录入时即校验格式 |
| **全电发票** | `InvoiceCode` 为空，`Amount` 为价税合计 | 系统根据号码长度(20位)自动识别全电票 |

### 3.5 错误码映射

| 错误码 | 含义 | 用户提示文案 | 系统处理 |
|:---|:---|:---|:---|
| `ResourceNotFound.NoInvoice` | 发票不存在 | ❌ 该发票在税务局系统中未找到，可能为假票 | **拦截**，标记为可疑 |
| `FailedOperation.InvoiceMismatch` | 发票数据不一致 | ❌ 发票信息与税局记录不匹配，请核对输入 | **拦截**，标记为异常 |
| `FailedOperation.CountLimitError` | 今日次数达限 | ⚠️ 该发票今日查验次数已达上限(5次)，请明日再试 | 缓存上次结果返回 |
| `ResourceUnavailable.TaxNetworkError` | 税务局网络异常 | ⚠️ 税务局系统繁忙，请稍后重试 | 加入重试队列 |
| `InvalidParameterValue.Ticket*` | 参数格式错误 | 请检查发票代码/号码/日期是否正确 | 前端校验拦截 |

### 4.2 数据库设计

#### 主表：invoice_verification_log（查验记录）

```sql
CREATE TABLE invoice_verification_log (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    
    -- 发票基本信息
    invoice_code    VARCHAR(20)   DEFAULT NULL    COMMENT '发票代码(全电票为空)',
    invoice_no      VARCHAR(20)   NOT NULL        COMMENT '发票号码',
    invoice_date    DATE          NOT NULL        COMMENT '开票日期',
    invoice_kind    VARCHAR(4)    DEFAULT NULL    COMMENT '票种类型代码',
    
    -- 查验请求信息
    check_code      VARCHAR(10)   DEFAULT NULL    COMMENT '校验码后6位',
    amount_input    DECIMAL(14,2) DEFAULT NULL    COMMENT '输入金额',
    source_type     VARCHAR(20)   NOT NULL        COMMENT '来源: purchase/sale/manual/batch/import',
    source_ref_id   BIGINT        DEFAULT NULL    COMMENT '关联的业务单据ID',
    
    -- 查验结果
    verify_status   TINYINT       NOT NULL        COMMENT '状态: 0待查/1通过/2异常/3拒绝/4超时',
    is_valid        BOOLEAN       DEFAULT NULL    COMMENT '真伪: true有效/false无效',
    is_abandoned    CHAR(1)       DEFAULT NULL    COMMENT '作废: Y/N',
    red_letter_mark BOOLEAN       DEFAULT NULL    COMMENT '红字标记',
    
    -- 税局返回的完整票面信息(JSON)
    tax_response    JSON          DEFAULT NULL    COMMENT '税局原始响应(完整JSON)',
    
    -- 标准化后的关键字段(冗余存储,方便查询)
    buyer_name      VARCHAR(200)  DEFAULT NULL    COMMENT '购方名称',
    buyer_tax_code  VARCHAR(30)   DEFAULT NULL    COMMENT '购方税号',
    seller_name     VARCHAR(200)  DEFAULT NULL    COMMENT '销方名称',
    seller_tax_code VARCHAR(30)   DEFAULT NULL    COMMENT '销方税号',
    amount_with_tax DECIMAL(14,2) DEFAULT NULL    COMMENT '价税合计',
    amount_no_tax   DECIMAL(14,2) DEFAULT NULL    COMMENT '不含税金额',
    tax_amount      DECIMAL(14,2) DEFAULT NULL    COMMENT '税额',
    
    -- 元数据
    api_request_id  VARCHAR(50)   DEFAULT NULL    COMMENT '腾讯云RequestID(用于问题追踪)',
    api_cost_ms     INT           DEFAULT NULL    COMMENT 'API耗时(ms)',
    error_code      VARCHAR(50)   DEFAULT NULL    COMMENT '错误码',
    error_message   VARCHAR(500)  DEFAULT NULL    COMMENT '错误信息',
    retry_count     TINYINT       DEFAULT 0       COMMENT '重试次数',
    
    -- 审计字段
    verified_by     VARCHAR(50)   DEFAULT NULL    COMMENT '操作人(system/user_id)',
    verified_at     DATETIME      NOT NULL        COMMENT '查验时间',
    created_at      DATETIME      DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME      DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    -- 索引
    INDEX idx_invoice_key (invoice_code, invoice_no),
    INDEX idx_verify_status (verify_status),
    INDEX idx_source (source_type, source_ref_id),
    INDEX idx_verified_at (verified_at),
    INDEX idx_seller_tax (seller_tax_code)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 
COMMENT='发票查验记录主表 - 每张发票每次查验一条记录';
```

### 5.1 单张发票查验流程


```python
class VerifyStatus:
    """发票查验状态枚举"""
    PENDING = 0      # 待查验（排队中）
    PASSED = 1       # ✅ 通过 — 发票真实且状态正常
    EXCEPTION = 2    # ⚠️ 异常 — 存在但有问题（作废/红冲/信息不一致）
    REJECTED = 3     # ❌ 拒绝 — 不存在或确认为无效发票
    TIMEOUT = 4      # ⏰ 超时 — API未及时响应，需重试


class ExceptionType:
    """异常类型枚举"""
    ABANDONED = "ABANDONED"           # 已作废
    RED_LETTER = "RED_LETTER"          # 红冲发票
    MISMATCH = "MISMATCH"              # 信息不一致
    DUPLICATE = "DUPLICATE"            # 重复报销风险
    AMOUNT_DIFF = "AMOUNT_DIFF"        # 金额不符
    TAX_CODE_INVALID = "TAX_INVALID"   # 税号异常
    EXPIRED = "EXPIRED"                # 超过查验期限


class RiskLevel:
    """风险等级"""
    LOW = 1      # 低风险 — 信息小偏差，可忽略
    MEDIUM = 2   # 中风险 — 需要财务人工审核
    HIGH = 3     # 高风险 — 可能涉及虚假发票
    CRITICAL = 4 # 严重 — 确认异常，必须阻止入账
```

### 7.1 核心调用封装

```python
"""
发票查验核心模块 - Dev_Dept ERP Finance Module
基于腾讯云 VatInvoiceVerifyNew 接口
"""

import json
import hashlib
import time
from datetime import datetime, timedelta
from typing import Optional, Dict, Any
from dataclasses import dataclass, field
from enum import IntEnum

# === 依赖说明 ===
# pip install tencentcloud-sdk-python-ocr
# 或: pip install tencentcloud-sdk-python
from tencentcloud.common.exception import TencentCloudSDKException
from tencentcloud.common.profile import ClientProfile
from tencentcloud.ocr.v20181119 import ocr_client, models


# ==================== 数据模型 ====================

class VerifyStatus(IntEnum):
    """查验状态"""
    PENDING = 0
    PASSED = 1
    EXCEPTION = 2
    REJECTED = 3
    TIMEOUT = 4


class RiskLevel(IntEnum):
    """风险等级"""
    LOW = 1
    MEDIUM = 2
    HIGH = 3
    CRITICAL = 4


@dataclass
class InvoiceInput:
    """发票查验输入"""
    invoice_no: str          # 发票号码 (必填)
    invoice_date: str        # 开票日期 YYYY-MM-DD (必填)
    invoice_code: str = ""   # 发票代码 (全电票为空)
    invoice_kind: str = ""   # 票种类型
    check_code: str = ""     # 校验码后6位
    amount: str = ""         # 金额 (专票=不含税, 全电=价税合计)
    region_code: str = ""    # 地区编码
    seller_tax_code: ""      # 销方税号
    source_type: str = "manual"  # 来源: purchase/sale/manual/batch
    source_ref_id: int = 0   # 关联业务单据ID


@dataclass
class VerifyResult:
    """查验结果"""
    status: VerifyStatus
    is_valid: Optional[bool] = None
    message: str = ""
    invoice_data: Dict[str, Any] = field(default_factory=dict)
    warnings: list = field(default_factory=list)
    request_id: str = ""
    cost_ms: int = 0
    result_source: str = ""  # API / CACHE / ERROR
    cached_at: Optional[datetime] = None


# ==================== 核心查验引擎 ====================

class InvoiceVerifier:
    """
    发票查验引擎
    
    使用方式:
        verifier = InvoiceVerifier(
            secret_id="your_secret_id",
            secret_key="your_secret_key"
        )
        result = verifier.verify(
            InvoiceInput(
                invoice_no="04138864",
                invoice_date="2025-01-15",
                invoice_code="144002288010",
                amount="498.90"
            )
        )
    """
    
    # 票种类型到必填参数的映射
    KIND_REQUIRED_FIELDS = {
        "01": ["amount"],       # 专票
        "03": ["amount"],       # 机动车
        "04": ["check_code"],   # 普票
        "08": ["amount"],       # 电子专票
        "10": ["check_code"],   # 电子普票
        "11": ["check_code"],   # 卷票
        "14": ["check_code"],   # 通行费
        "15": ["amount"],       # 二手车
        "16": ["amount"],       # 财政票据
    }
    
    def __init__(
        self,
        secret_id: str,
        secret_key: str,
        region: str = "ap-guangzhou",
        timeout: int = 10,
        cache_ttl_hours: int = 24,
    ):
        self.secret_id = secret_id
        self.secret_key = secret_key
        self.region = region
        self.timeout = timeout
        self.cache_ttl = timedelta(hours=cache_ttl_hours)
        
        # 初始化腾讯云客户端
        client_profile = ClientProfile()
        client_profile.httpProfile.endpoint = "ocr.tencentcloudapi.com"
        client_profile.httpProfile.reqTimeout = timeout
        self.client = ocr_client.OcrClient(
            credential={"secretId": secret_id, "secretKey": secret_key},
            region=region,
            profile=client_profile
        )
        
        # 内存缓存 (生产环境应替换为Redis)
        self._cache: Dict[str, tuple] = {}  # key -> (result_json, timestamp)
    
    def _make_cache_key(self, inp: InvoiceInput) -> str:
        """生成缓存key"""
        raw = f"{inp.invoice_code}|{inp.invoice_no}|{inp.invoice_date}"
        return hashlib.md5(raw.encode()).hexdigest()
    
    def _get_from_cache(self, cache_key: str) -> Optional[Dict]:
        """从缓存获取结果"""
        if cache_key in self._cache:
            result_data, ts = self._cache[cache_key]
            if datetime.now() - ts < self.cache_ttl:
                return result_data
            else:
                del self._cache[cache_key]
        return None
    
    def _set_cache(self, cache_key: str, data: Dict):
        """写入缓存"""
        self._cache[cache_key] = (data, datetime.now())
    
    def _pre_validate(self, inp: InvoiceInput) -> tuple[bool, str]:
        """
        本地预校验
        Returns: (is_valid, error_message)
        """
        # 1. 发票号码校验
        if not inp.invoice_no or len(inp.invoice_no) < 8:
            return False, f"发票号码格式错误: '{inp.invoice_no}'，应为8位或20位"
        
        # 2. 日期格式校验
        try:
            dt = datetime.strptime(inp.invoice_date, "%Y-%m-%d")
            if dt > datetime.now():
                return False, "开票日期不能晚于今天"
            if (datetime.now() - dt).days > 365 * 5:
                return False, "仅支持查验5年内的发票"
        except ValueError:
            return False, f"日期格式错误: '{inp.invoice_date}'，应为YYYY-MM-DD"
        
        # 3. 票种特定字段校验
        if inp.invoice_kind:
            required = self.KIND_REQUIRED_FIELDS.get(inp.invoice_kind, [])
            for field in required:
                value = getattr(inp, field, "")
                if not value:
                    field_names = {
                        "amount": "不含税金额",
                        "check_code": "校验码后6位"
                    }
                    return False, f"{inp.invoice_kind}型发票需要填写{field_names.get(field, field)}"
        
        return True, ""
    
    def _parse_api_result(self, response: Dict) -> VerifyResult:
        """解析API响应为标准化结果"""
        resp_data = response.get("Response", {})
        invoice = resp_data.get("Invoice", {})
        request_id = resp_data.get("RequestId", "")
        
        # 基础信息提取
        result = VerifyResult(
            status=VerifyStatus.PASSED,
            is_valid=True,
            request_id=request_id,
            result_source="API",
            invoice_data={
                "code": invoice.get("Code", ""),
                "number": invoice.get("Number", ""),
                "date": invoice.get("Date", ""),
                "buyer_name": invoice.get("BuyerName", ""),
                "buyer_tax_code": invoice.get("BuyerTaxCode", ""),
                "seller_name": invoice.get("SellerName", ""),
                "seller_tax_code": invoice.get("SellerTaxCode", ""),
                "amount_with_tax": invoice.get("AmountWithTax", ""),
                "amount_without_tax": invoice.get("AmountWithoutTax", ""),
                "tax_amount": invoice.get("TaxAmount", ""),
                "type": invoice.get("Type", ""),
            }
        )
        
        # 状态判断
        is_abandoned = invoice.get("IsAbandoned", "N") == "Y"
        red_letter = invoice.get("RedLetterInvoiceMark", False)
        
        warnings = []
        if is_abandoned:
            result.status = VerifyStatus.EXCEPTION
            result.is_valid = False
            warnings.append({"type": "ABANDONED", "msg": "该发票已作废"})
        if red_letter:
            result.status = VerifyStatus.EXCEPTION
            warnings.append({"type": "RED_LETTER", "msg": "该发票已被红冲"})
        
        result.warnings = warnings
        return result
    
    def verify(self, inp: InvoiceInput, force_refresh: bool = False) -> VerifyResult:
        """
        执行发票查验 (主入口)
        
        Args:
            inp: 发票输入信息
            force_refresh: 强制刷新缓存
        
        Returns:
            VerifyResult: 查验结果
        """
        start_time = time.time()
        
        # Step 1: 本地预校验
        valid, err_msg = self._pre_validate(inp)
        if not valid:
            return VerifyResult(
                status=VerifyStatus.REJECTED,
                is_valid=False,
                message=f"预校验失败: {err_msg}",
                result_source="VALIDATION_ERROR"
            )
        
        # Step 2: 查询缓存
        cache_key = self._make_cache_key(inp)
        if not force_refresh:
            cached = self._get_from_cache(cache_key)
            if cached:
                result = VerifyResult(
                    status=VerifyStatus.PASSED,
                    is_valid=True,
                    result_source="CACHE",
                    invoice_data=cached,
                    cached_at=self._cache[cache_key][1]
                )
                result.cost_ms = int((time.time() - start_time) * 1000)
                return result
        
        # Step 3: 调用腾讯云API
        try:
            req = models.VatInvoiceVerifyNewRequest()
            req.FromJsonString(json.dumps({
                "InvoiceCode": inp.invoice_code or None,
                "InvoiceNo": inp.invoice_no,
                "InvoiceDate": inp.invoice_date,
                "InvoiceKind": inp.invoice_kind or None,
                "CheckCode": inp.check_code or None,
                "Amount": inp.amount or None,
                "RegionCode": inp.region_code or None,
                "SellerTaxCode": inp.seller_tax_code or None,
                "EnableCommonElectronic": bool(inp.seller_tax_code and inp.region_code),
                "EnableTodayInvoice": True,  # 允许查验当日发票
            }))
            
            resp = self.client.VatInvoiceVerifyNew(req)
            response_dict = json.loads(resp.to_json_string())
            
            # 解析结果
            result = self._parse_api_result(response_dict)
            result.cost_ms = int((time.time() - start_time) * 1000)
            
            # 写入缓存
            if result.status == VerifyStatus.PASSED:
                self._set_cache(cache_key, result.invoice_data)
            
            return result
            
        except TencentCloudSDKException as e:
            error_code = e.get_code()
            error_msg = e.get_message()
            
            # 错误码分类处理
            if error_code == "ResourceNotFound.NoInvoice":
                return VerifyResult(
                    status=VerifyStatus.REJECTED,
                    is_valid=False,
                    message=f"发票不存在: {error_msg}",
                    request_id="",
                    result_source="API_ERROR"
                )
            elif error_code == "FailedOperation.InvoiceMismatch":
                return VerifyResult(
                    status=VerifyStatus.EXCEPTION,
                    is_valid=False,
                    message=f"发票信息不匹配: {error_msg}",
                    request_id="",
                    result_source="API_ERROR"
                )
            elif error_code == "FailedOperation.CountLimitError":
                # 尝试返回缓存中的旧结果
                old_cached = self._get_from_cache(cache_key.replace("_forced", ""))
                if old_cached:
                    return VerifyResult(
                        status=VerifyStatus.PASSED,
                        is_valid=True,
                        message="今日查验次数已达上限,返回最近一次结果",
                        result_source="CACHE_FALLBACK",
                        invoice_data=old_cached
                    )
                return VerifyResult(
                    status=VerifyStatus.TIMEOUT,
                    message=f"查验次数达上限且无缓存: {error_msg}",
                    result_source="API_LIMIT"
                )
            else:
                return VerifyResult(
                    status=VerifyStatus.TIMEOUT,
                    message=f"API调用异常 [{error_code}]: {error_msg}",
                    result_source="API_ERROR"
                )
        except Exception as e:
            return VerifyResult(
                status=VerifyStatus.TIMEOUT,
                message=f"系统异常: {str(e)}",
                result_source="SYSTEM_ERROR"
            )
```
