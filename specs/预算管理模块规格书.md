## 一、模块概述

预算管理是企业财务管控的核心工具，通过"编-审-控-调-分析"闭环，实现资金的事前计划、事中控制、事后分析。

### 1.1 核心价值

| 价值点 | 说明 |
|--------|------|
| 资金可视化 | 事前规划资金使用，避免资金黑洞 |
| 过程管控 | 事中控制预算执行，超限预警/阻断 |
| 绩效考核 | 部门/项目维度预算执行率考核 |
| 决策支撑 | 实时预算执行分析，支撑经营决策 |

### 1.2 对标金蝶云星空

金蝶云星空预算管理模块功能：

- **预算编制**：支持年度/季度/月度多周期预算编制
- **预算审批**：多级审批流程，支持预算冻结/解冻
- **预算控制**：严格控制/弹性控制/不控制三种模式
- **预算调整**：支持预算调整申请与审批
- **预算分析**：执行率分析、预算与实际对比、滚动预测
- **预算考核**：部门/项目维度预算执行率考核

## 三、详细功能说明

### 3.1 预算基础设置

#### 3.1.1 预算周期定义

| 字段 | 说明 | 示例 |
|------|------|------|
| 周期类型 | 年度/季度/月度 | 月度 |
| 开始月份 | 预算周期起始月 | 1月 |
| 结束月份 | 预算周期结束月 | 12月 |
| 编制提前期 | 提前多少天开始编制 | 30天 |
| 锁定规则 | 何时锁定不可修改 | 审批通过后 |

#### 3.1.2 预算组织架构

```
组织层级：
├── 公司级预算
├── 事业部/部门级预算
│   ├── 销售部
│   ├── 采购部
│   ├── 生产部
│   └── 管理部门
└── 项目级预算
    ├── 项目A
    └── 项目B
```

#### 3.1.3 预算科目体系

| 科目类型 | 方向 | 说明 |
|----------|------|------|
| 收入类 | 预算增加 | 销售收入、其他收入 |
| 成本类 | 预算减少 | 直接成本、制造费用 |
| 费用类 | 预算减少 | 管理费用、销售费用 |
| 资产类 | 预算变动 | 固定资产采购、在建工程 |

### 3.2 预算编制

#### 3.2.1 年度预算编制流程

```
开始 -> 选择预算年度 -> 选择编制部门 -> 选择预算模板
  -> 按模板填报数据 -> 数据校验 -> 提交审批 -> 结束
```

#### 3.2.2 预算编制规则

| 规则类型 | 说明 | 配置方式 |
|----------|------|----------|
| 零基预算 | 从零开始编制 | 每年度重置 |
| 增量预算 | 基于上年调整 | 增量比例配置 |
| 弹性预算 | 基于业务量调整 | 公式定义 |

#### 3.2.3 预算数据来源

| 来源 | 说明 |
|------|------|
| 手工录入 | 直接在系统填写 |
| Excel导入 | 模板下载-填写-导入 |
| 历史数据 | 基于上年实际/预算复制 |
| 业务计划 | 关联销售计划、生产计划 |

### 3.3 预算审批

#### 3.3.1 审批流程

```
编制人提交 -> 部门主管审批 -> 财务经理审批 -> 财务总监审批 -> 发布
     ↓              ↓              ↓
   驳回          驳回          驳回
     ↓              ↓              ↓
  返回编制      返回编制      返回编制
```

#### 3.3.2 审批权限矩阵

| 预算金额 | 审批层级 | 审批人 |
|----------|----------|--------|
| < 10万 | 1级 | 部门主管 |
| 10-50万 | 2级 | 部门主管 + 财务经理 |
| 50-100万 | 3级 | 部门主管 + 财务经理 + 财务总监 |
| >= 100万 | 4级 | + 总经理 |

### 3.4 预算控制

#### 3.4.1 控制模式

| 控制模式 | 说明 | 触发动作 |
|----------|------|----------|
| 严格控制 | 不得超预算 | 直接拒绝报销 |
| 弹性控制 | 可超一定比例 | 预警 + 审批升级 |
| 不控制 | 仅记录分析 | 仅记录，无拦截 |

#### 3.4.2 控制维度

```
预算控制维度：
├── 维度1：部门维度
│   └── 各部门独立控制
├── 维度2：项目维度
│   └── 各项目独立控制
├── 维度3：费用项维度
│   └── 各费用项独立控制
└── 维度4：时间维度
    └── 月度/季度/年度控制
```

#### 3.4.3 预警规则

| 预警级别 | 触发条件 | 通知方式 |
|----------|----------|----------|
| L1 提醒 | 达到 70% | 站内信 |
| L2 预警 | 达到 90% | 站内信 + 邮件 |
| L3 警告 | 达到 100% | 全渠道通知 |
| L4 超限 | 超过 100% | 全渠道 + 阻断审批 |

### 3.5 预算调整

#### 3.5.1 调整类型

| 调整类型 | 说明 |
|----------|------|
| 科目间调整 | A费用项调减，B费用项调增 |
| 部门间调整 | A部门调减，B部门调增 |
| 时间维度调整 | Q1调减，Q2调增 |
| 追加预算 | 整体追加，需额外审批 |

#### 3.5.2 调整流程

```
调整申请 -> 填写调整金额 -> 选择调整原因 -> 提交审批
  -> 财务经理审批 -> 财务总监审批（>50万）-> 执行调整
```

### 3.6 预算分析

#### 3.6.1 分析报表

| 报表名称 | 说明 |
|----------|------|
| 预算执行进度表 | 各部门/项目预算执行率 |
| 预算与实际对比表 | 预算vs实际vs差异 |
| 预算差异分析表 | 差异原因分析 |
| 滚动预测表 | 基于执行数据的未来预测 |
| 预算考核表 | 各部门考核结果 |

#### 3.6.2 分析维度

```
分析维度：
├── 时间维度：年度/季度/月度
├── 组织维度：公司/事业部/部门/项目
├── 科目维度：收入/成本/费用
└── 对比维度：预算/实际/差异/执行率
```

## 五、业务流程图

详见配套流程图文件：`预算管理总流程图.html`

### 5.1 核心流程

```
1. 预算编制流程
   启动编制 -> 模板选择 -> 数据填报 -> 校验 -> 提交审批 -> 发布

2. 预算控制流程
   报销申请 -> 预算检查 -> 额度判断 -> 通过/预警/拒绝 -> 执行/调整

3. 预算调整流程
   调整申请 -> 原因填写 -> 审批 -> 执行调整 -> 更新台账

4. 预算分析流程
   数据采集 -> 执行率计算 -> 差异分析 -> 生成报表 -> 考核评价
```

## 八、实施建议

### 8.1 实施步骤

```
Phase 1: 基础设置（1-2周）
  - 组织架构定义
  - 科目体系定义
  - 预算模板配置

Phase 2: 编制上线（2-3周）
  - 历史数据迁移
  - 用户培训
  - 试运行

Phase 3: 控制启用（1-2周）
  - 控制规则配置
  - 预警规则配置
  - 正式启用

Phase 4: 分析深化（持续）
  - 分析模型优化
  - 考核体系完善
```

### 8.2 关键成功因素

| 因素 | 说明 |
|------|------|
| 领导支持 | 高层支持是预算管理成功的关键 |
| 数据质量 | 基础数据准确是预算有效的前提 |
| 持续优化 | 根据执行情况持续调整预算模型 |
| 全员参与 | 各部门参与才能保证预算落地 |
