## 一、业务背景

### 1.1 问题描述

在与越南、泰国等海外供应商对账时，经常会出现品质扣款、迟延罚款等情况。传统做法是从应付账款中直接扣减，但这些国家的税法有特殊规定：

> **核心约束**：发票已开具、双方已对账确认的金额，不能再从应付账款中扣减。

因此，必须通过**现金收/付款项**的方式单独处理扣款项。

### 1.2 业务场景

| 场景 | 说明 | 扣款方向 |
|------|------|----------|
| 品质扣款 | 货物质量问题，供应商赔款 | 供应商→我方 |
| 迟延罚款 | 供应商延期交付，被罚款 | 供应商→我方 |
| 我方补偿 | 我方责任，主动赔偿供应商 | 我方→供应商 |
| 其他补偿 | 价格调整、业务补偿等 | 双方均可 |

## 二、功能范围

### 2.1 核心功能

| 功能编号 | 功能名称 | 功能说明 | 优先级 |
|----------|----------|----------|--------|
| F01 | 现金扣款申请 | 发起现金扣款单，关联原应付单据 | P0 |
| F02 | 扣款审核 | 财务人员审核扣款事项 | P0 |
| F03 | 现金收/付款确认 | 确认现金收付，更新账户余额 | P0 |
| F04 | 往来核销 | 核销应付/应收，生成核销记录 | P0 |
| F05 | 凭证生成 | 生成总账凭证，传递到凭证引擎 | P0 |
| F06 | 扣款查询 | 按供应商/日期/状态查询扣款记录 | P1 |
| F07 | 统计报表 | 按供应商/月份/原因统计扣款 | P2 |

### 2.2 功能边界

**本模块负责：**
- 现金扣款的申请、审核、确认
- 与应付管理的联动
- 凭证模板的生成
- 往来核销处理

**本模块不负责：**
- 采购对账逻辑（由采购模块处理）
- 银行转账处理（由现金管理模块处理）
- 发票开具/作废（由发票模块处理）

## 三、业务流程

### 3.1 完整流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    现金扣款处理完整流程                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ① 采购对账发现扣款项                                             │
│         ↓                                                        │
│  ② 发起现金扣款申请                                               │
│         ↓                                                        │
│  ③ 财务审核确认                                                   │
│         ↓                                                        │
│  ④ 生成现金扣款单                                                 │
│         ↓                                                        │
│  ⑤ 现金收/付款确认                                               │
│         ↓                                                        │
│  ⑥ 往来核销处理                                                   │
│         ↓                                                        │
│  ⑦ 生成凭证                                                      │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 流程详细说明

#### 步骤1：采购对账发现扣款项
- 采购人员与供应商对账
- 发现品质问题/迟延等需要扣款的情况
- 评估扣款金额是否需要走现金流程

#### 步骤2：发起现金扣款申请
- 采购人员进入系统，发起"现金扣款申请"
- 填写信息：
  - 供应商（必填）
  - 扣款方向（供应商→我方 / 我方→供应商）
  - 扣款金额（必填）
  - 扣款原因（必填）
  - 关联单据（必填，原应付发票）
- 上传扣款协议（如有）
- 提交审核

#### 步骤3：财务审核
- 财务人员收到审核通知
- 审核内容：
  - 扣款事由是否合理
  - 金额是否正确
  - 附件是否齐全
- 通过/驳回

#### 步骤4：生成现金扣款单
- 审核通过后，系统自动生成"现金扣款单"
- 分配唯一单据编号（格式：CASH-YYYYMMDD-XXX）
- 状态更新为"待确认"

#### 步骤5：现金收/付款确认
- 财务人员执行收/付款操作
- 选择支付方式：现金/银行转账
- 确认收/付款信息
- 提交确认

#### 步骤6：往来核销
- 系统自动进行往来核销
- 更新供应商应付/应收余额
- 生成核销记录

#### 步骤7：生成凭证
- 根据凭证模板生成凭证分录
- 传递到总账凭证引擎
- 完成整个流程

## 四、数据模型

### 4.1 核心表：cash_deduction

```sql
CREATE TABLE cash_deduction (
    -- 主键
    id              BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    deduction_no    VARCHAR(50) UNIQUE NOT NULL COMMENT '扣款单编号',
    
    -- 基本信息
    deduction_date  DATE NOT NULL COMMENT '扣款日期',
    supplier_id     BIGINT NOT NULL COMMENT '供应商ID',
    currency        VARCHAR(10) NOT NULL COMMENT '币种',
    
    -- 扣款信息
    direction       TINYINT NOT NULL COMMENT '扣款方向:1=供应商→我方,2=我方→供应商',
    amount          DECIMAL(18,4) NOT NULL COMMENT '扣款金额',
    reason          VARCHAR(50) NOT NULL COMMENT '扣款原因',
    reason_remark   VARCHAR(200) COMMENT '扣款原因备注',
    
    -- 关联信息
    source_type     VARCHAR(20) COMMENT '来源单据类型',
    source_id       BIGINT COMMENT '来源单据ID',
    original_amount DECIMAL(18,4) COMMENT '原应付金额',
    
    -- 核销信息
    writeoff_status TINYINT DEFAULT 0 COMMENT '核销状态:0=未核销,1=已核销',
    writeoff_time   DATETIME COMMENT '核销时间',
    
    -- 支付信息
    pay_method      VARCHAR(20) COMMENT '支付方式',
    pay_account     VARCHAR(50) COMMENT '收付款账号',
    pay_status      TINYINT DEFAULT 0 COMMENT '支付状态:0=未支付,1=已支付',
    pay_time        DATETIME COMMENT '支付时间',
    
    -- 凭证信息
    voucher_id      BIGINT COMMENT '凭证ID',
    
    -- 审核信息
    status          TINYINT DEFAULT 0 COMMENT '状态:0=草稿,1=审核中,2=待确认,3=已完成,4=已驳回',
    create_by       BIGINT NOT NULL COMMENT '创建人',
    create_time     DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_by       BIGINT COMMENT '更新人',
    update_time     DATETIME COMMENT '更新时间',
    audit_by        BIGINT COMMENT '审核人',
    audit_time      DATETIME COMMENT '审核时间',
    audit_remark    VARCHAR(200) COMMENT '审核意见',
    
    -- 附件
    attachment_ids  VARCHAR(500) COMMENT '附件ID列表(JSON)',
    
    -- 扩展字段
    ext_data        TEXT COMMENT '扩展数据(JSON)',
    
    -- 逻辑删除
    deleted         TINYINT DEFAULT 0 COMMENT '删除标记'
) COMMENT '现金扣款单表';
```

### 4.2 核销记录表：cash_deduction_writeoff

```sql
CREATE TABLE cash_deduction_writeoff (
    id                  BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    deduction_id        BIGINT NOT NULL COMMENT '扣款单ID',
    
    -- 核销信息
    writeoff_type       TINYINT NOT NULL COMMENT '核销类型:1=收款核销,2=付款核销',
    writeoff_amount     DECIMAL(18,4) NOT NULL COMMENT '核销金额',
    
    -- 关联信息
    ap_id               BIGINT COMMENT '应付单ID',
    ar_id               BIGINT COMMENT '应收单ID',
    
    -- 凭证信息
    voucher_id          BIGINT COMMENT '凭证ID',
    voucher_no          VARCHAR(50) COMMENT '凭证号',
    
    -- 审计字段
    create_by           BIGINT NOT NULL COMMENT '创建人',
    create_time         DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    
    INDEX idx_deduction (deduction_id),
    INDEX idx_ap (ap_id),
    INDEX idx_voucher (voucher_id)
) COMMENT '现金扣款核销记录表';
```

## 五、凭证模板

### 5.1 模板A：供应商→我方（我们收钱）

| 项目 | 内容 |
|------|------|
| 模板代码 | CASH_DEDUCT_A |
| 模板名称 | 供应商赔款收款 |
| 业务描述 | 供应商因品质问题/迟延等赔款给我们 |
| 借方科目 | 1001（库存现金）/ 1002（银行存款） |
| 贷方科目 | 2202（应付账款-XX供应商） |
| 辅助核算 | 供应商、部门、项目 |

**示例分录：**
```
借：银行存款   1,000.00
贷：应付账款-XX供应商  1,000.00

摘要：收到XX供应商品质扣款补偿
辅助：供应商=XX公司，部门=采购部
```

### 5.2 模板B：我方→供应商（我们付钱）

| 项目 | 内容 |
|------|------|
| 模板代码 | CASH_DEDUCT_B |
| 模板名称 | 我方赔款付款 |
| 业务描述 | 我们因自身原因赔款给供应商 |
| 借方科目 | 2202（应付账款-XX供应商） |
| 贷方科目 | 1001（库存现金）/ 1002（银行存款） |
| 辅助核算 | 供应商、部门、项目 |

**示例分录：**
```
借：应付账款-XX供应商  500.00
贷：银行存款    500.00

摘要：支付XX供应商迟延补偿款
辅助：供应商=XX公司，部门=生产部
```

| 字段分类 | 字段名称 | 字段类型 | 必填 | 默认值 | 说明 |

| 基本信息 | 扣款单编号 | 文本 | - | 自动 | 系统自动生成 |

| 关联信息 | 来源单据 | 关联选择 | ✅ | - | 弹出选择框 |
| 关联信息 | 原应付金额 | 数字 | - | - | 自动填充 |
| 支付信息 | 支付方式 | 下拉 | ✅ | - | 现金/银行转账 |
| 支付信息 | 收付款账号 | 下拉 | ✅ | - | 从账户列表选择 |
| 附件 | 扣款协议 | 文件上传 | ⚠️ | - | 建议上传PDF/图片 |
| 附件 | 其他附件 | 文件上传 | - | - | 可上传多个 |
| 审核信息 | 审核意见 | 文本 | - | - | 审核人填写 |
| 系统信息 | 单据状态 | 文本 | - | - | 只读显示 |

### 6.2 字段校验规则

| 字段 | 校验规则 |
|------|----------|
| 扣款金额 | > 0；≤ 原应付金额（如果设置了上限） |
| 供应商 | 必须是有效供应商 |
| 扣款日期 | ≤ 当前日期 |
| 扣款方向 | 必选 |
| 扣款原因 | 必选；选择"其他"时，备注必填 |
| 来源单据 | 必须是该供应商的有效应付单据 |

## 十、非功能性需求

### 10.1 性能要求

| 指标 | 要求 |
|------|------|
| 单据查询响应时间 | ≤ 500ms |
| 单据提交响应时间 | ≤ 1s |
| 批量导出 | 支持导出1000条记录 |

### 10.2 安全要求

- 所有操作记录操作日志
- 敏感操作（如删除）需二次确认
- 审核记录永久保存，不可删除

### 10.3 各国差异适配

| 国家 | 特殊要求 | 实现方式 |
|------|----------|----------|
| 越南 | 必须通过现金处理 | 必填扣款协议附件 |
| 泰国 | 需双方书面确认 | 支持附件上传 |
| 菲律宾 | 无特殊要求 | 正常流程 |
| 沙特 | VAT合规 | 与VAT模块联动 |
