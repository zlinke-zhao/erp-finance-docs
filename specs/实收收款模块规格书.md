## 1. 模块定位与业务目标

### 1.1 一句话定义

**实收收款模块是ERP系统中"资金流入"的核心枢纽，负责从银行资金到账开始，经过收款登记、应收核销、凭证生成，最终写入总账的全流程管理。**

### 1.2 在整体流程中的位置

```
销售订单 → 送货单 → 应收对账 → 税务开票 → 发票录入 ─┐
                                                    │
采购申请 → 采购订单 → 收货验收 → 应付对账 ──────────┤
                                                    ▼
                                            ┌──────────────┐
                                            │ 🔄 实收收款   │ ◄── 本模块
                                            │ (本规格书)    │
                                            └──────┬───────┘
                                                   │
                                                   ▼
                                            ┌──────────────┐
                                            │ 🧾 总账凭证    │ ← P0待建
                                            │              │
                                            │ 借:银行存款   │
                                            │ 贷:应收账款   │
                                            └──────────────┘
```

### 1.3 核心业务目标

| 目标 | 指标 | 当前痛点 |
|:---|:---|:---|
| **收款可追溯** | 每笔收款100%关联到具体应收单据 | 手工Excel对账，容易漏记 |
| **核销准确率** | 自动核销匹配率≥90%，人工确认率≤10% | 全靠人眼逐笔核对 |
| **账龄准确性** | 应收余额与实际回款实时同步 | 月末才调一次，平时不准 |
| **预收透明度** | 预收余额实时可见、自动抵扣 | 预收款记在脑子里或散落各处 |
| **凭证自动化** | 收款审核通过后自动生成凭证 | 手工做分录，容易出错 |
| **到账时效** | 从银行到账到系统登记≤4小时 | 发现晚、登记不及时 |

### 1.4 用户角色

| 角色 | 权限范围 | 典型操作 |
|:---|:---|:---|
| **财务会计** | 创建/编辑/删除收款单 | 录入收款、核销操作、查看报表 |
| **财务主管** | 审核所有收款单、审批异常 | 审核、红冲、权限审批 |
| **出纳** | 仅录入和查询 | 登记到账信息（不可核销） |
| **销售经理** | 只读查看自己客户的收款情况 | 回款进度、客户欠款 |
| **系统管理员** | 配置科目模板、核销规则、银行账户 | 参数配置 |

#### F07. 供应商现金扣款收款（集成自研供应商扣款审批流程）

> **业务背景**：越南/泰国税法规定，发票已开具且双方对账确认的金额，不能再从应付账款中直接扣减。
> 因此当发生供应商品质扣款/迟延罚款时，不走应付对账或红字发票，而是由供应商以**现金**形式缴纳扣款，
> 本模块负责登记该笔现金收款，并关联自研系统的供应商扣款审批单形成闭环。
> **此场景不与税务模块对账**，凭证独立入总账。

| 功能点 | 描述 | 优先级 |
|:---|:---|:---:|
| **F07-01** 扣款收款登记 | 创建收款单时，来源类型选"供应商扣款"，必填供应商、扣款单号、扣款原因、金额 | P0 |
| **F07-02** 扣款单号关联 | 必须填入自研系统供应商扣款审批流程生成的扣款单号，用于冲销闭环 | P0 |
| **F07-03** 独立核销逻辑 | 不走应收核销引擎（F02），审核通过后直接标记"已冲销"（关联扣款单号即为冲销依据） | P0 |
| **F07-04** 独立凭证模板 | 借:1001库存现金 贷:6301营业外收入-供应商赔偿，不经过应付账款科目 | P0 |
| **F07-05** 扣款原因分类 | 品质扣款 / 迟延罚款 / 其他，必选，影响凭证摘要描述 | P0 |
| **F07-06** 扣款查询 | 按供应商/扣款原因/日期/状态查询扣款收款记录 | P1 |
| **F07-07** 扣款统计报表 | 按供应商/月份/原因汇总扣款金额 | P2 |

**功能边界：**
- ✅ 仅处理供应商→我方（现金收款），不支持我方→供应商方向
- ✅ 仅支持现金（CASH）收款方式
- ✅ 核销对象为自研系统"供应商扣款审批流程"的扣款单号，非应付对账
- ❌ 不与税务模块对账，不开红字发票，不影响应付余额
- ❌ 凭证不入应付账款科目，直接入营业外收入

### 3.2 数据库DDL

#### 表1: receipt（收款单主表）

```sql
CREATE TABLE receipt (
    id                BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    
    -- === 基本信息 ===
    receipt_no        VARCHAR(20)  NOT NULL UNIQUE  COMMENT '收款单编号(自动生成)',
    customer_id       BIGINT       NOT NULL        COMMENT '客户ID',
    
    -- === 金额信息 ===
    amount            DECIMAL(14,2) NOT NULL       COMMENT '收款金额(原币)',
    currency          VARCHAR(6)   NOT NULL DEFAULT 'CNY' COMMENT '币种',
    exchange_rate     DECIMAL(12,6) DEFAULT 1.000000 COMMENT '汇率',
    amount_cny        DECIMAL(14,2) NOT NULL        COMMENT '折合人民币金额',
    unwriteoff_amount DECIMAL(14,2) NOT NULL        COMMENT '未核销余额(初始=amount,核销后递减)',
    
    -- === 收款信息 ===
    received_date     DATE         NOT NULL        COMMENT '收款日期',
    payment_method    VARCHAR(20)  NOT NULL        COMMENT '收款方式: BANK_TRANSFER/CASH/POS/ALIPAY/WECHAT/BILL/DRAFT/OTHER',
    bank_account_id   BIGINT       DEFAULT NULL     COMMENT '收款银行账户ID',
    bank_ref_no       VARCHAR(50)  DEFAULT NULL     COMMENT '银行参考号/流水号',
    
    -- === 来源信息 ===
    source_type       VARCHAR(20)  NOT NULL DEFAULT 'MANUAL' COMMENT '来源: MANUAL/BANK_IMPORT/BATCH/API',
    source_ref        VARCHAR(50)  DEFAULT NULL     COMMENT '来源引用(银行流水ID等)',
    
    -- === 供应商扣款信息（仅 source_type=DEDUCTION 时有值） ===
    deduction_no      VARCHAR(50)  DEFAULT NULL     COMMENT '供应商扣款单号(关联自研供应商扣款审批流程)',
    supplier_id       BIGINT       DEFAULT NULL     COMMENT '供应商ID(仅扣款场景)',
    deduction_reason  VARCHAR(20)  DEFAULT NULL     COMMENT '扣款原因: QUALITY/DELAY/OTHER',
    deduction_remark  VARCHAR(200) DEFAULT NULL     COMMENT '扣款原因说明',
    
    -- === 关联信息 ===
    voucher_id        BIGINT       DEFAULT NULL     COMMENT '生成的凭证ID(关联总账模块)',
    
    -- === 附件与备注 ===
    attachment_url    VARCHAR(500) DEFAULT NULL     COMMENT '银行回单附件URL',
    remark            VARCHAR(500) DEFAULT NULL     COMMENT '备注',
    
    -- === 状态 ===
    status            VARCHAR(16)  NOT NULL DEFAULT 'DRAFT' COMMENT 'DRAFT/VERIFIED/CANCELLED',
    
    -- === 审计 ===
    created_by        BIGINT       DEFAULT NULL     COMMENT '创建人ID',
    created_at        DATETIME     DEFAULT CURRENT_TIMESTAMP,
    updated_by        BIGINT       DEFAULT NULL     COMMENT '最后修改人',
    updated_at        DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    verified_by       BIGINT       DEFAULT NULL     COMMENT '审核人',
    verified_at       DATETIME     DEFAULT NULL     COMMENT '审核时间',
    
    -- === 索引 ===
    INDEX idx_customer_date (customer_id, received_date),
    INDEX idx_status (status),
    INDEX idx_source (source_type, source_ref),
    INDEX idx_deduction (deduction_no),
    INDEX idx_supplier (supplier_id, received_date),
    INDEX idx_received_date (received_date)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 
COMMENT='收款单主表 - 记录每一笔实收款项';
```

#### 表2: receipt_writeoff（核销记录表）

```sql
CREATE TABLE receipt_writeoff (
    id                BIGINT PRIMARY KEY AUTO_INCREMENT,
    
    -- === 关联 ===
    receipt_id        BIGINT       NOT NULL COMMENT '收款单ID',
    ar_invoice_id     BIGINT       NOT NULL COMMENT '应收单ID',
    so_ref_id         BIGINT       DEFAULT NULL COMMENT '销售订单ID(冗余,方便查询)',
    
    -- === 核销金额 ===
    writeoff_amount   DECIMAL(14,2) NOT NULL       COMMENT '本次核销金额',
    prepay_used       DECIMAL(14,2) DEFAULT 0      COMMENT '其中使用预收款的金额',
    discount_amount   DECIMAL(14,2) DEFAULT 0      COMMENT '折扣/坏账金额(若有)',
```

## 4. 业务规则与凭证模板

### 4.1 七种收款场景的会计分录

| 场景代码 | 场景描述 | 借方 | 贷方 | 备注 |
|:---:|:---|:---|:---|:---|
| **S01** | 正常收款核销 | `1002 银行存款-xx银行` | `1122 应收账款-XX客户` | 最常见的标准场景 |
| **S02** | 纯预收款（无订单） | `1002 银行存款-xx银行` | `2203 预收账款-XX客户` | 先款后货/VIP存入 |
| **S03** | 预收转应收核销 | `2203 预收账款-XX客户` | `1122 应收账款-XX客户` | 新订单产生时的内部转账 |
| **S04** | 收款+提前付款折扣 | `1002 银行存款` + `6603 财务费用-折扣支出` | `1122 应收账款` | 客户享早鸟折扣 |
| **S05** | 收到承兑汇票 | `1121 应收票据-承兑汇票` | `1122 应收账款-XX客户` | 以票据代替现金 |
| **S06** | 汇票贴现/兑现 | `1002 银行存款` + `6603 财务费用-贴现息` | `1121 应收票据-承兑汇票` | 票据到期或贴现 |
| **S07** | 供应商现金扣款收款 | `1001 库存现金` | `6301 营业外收入-供应商赔偿` | F07场景，不经过应付/应收 |

> **S07 与 S01~S06 的关键区别**：
> - S07 的收款来源是**供应商**（不是客户），科目走**营业外收入**（不走应收账款/应付账款）
> - S07 不与税务模块对账，不影响进项发票/销项发票状态
> - S07 的核销依据是**供应商扣款单号**（关联自研审批流程），不走应收核销引擎
> - S07 仅支持现金收款方式

### 4.3 收款单编号规则

```
格式: SK + 年月 + 5位序号
示例: SK20260400001 ~ SK20260499999

规则:
• SK = ShouKou(收款)缩写
• 按月份重置序号
• 删除后不填补空号(保持连续性)
• 作废后原号作废,新生成新号
```

### 4.4 收款方式枚举

| code | 名称 | 需填字段 | 特殊处理 |
|:---|:---|:---|:---|
| `BANK_TRANSFER` | 银行转账 | 银行账户、流水号 | 可对接银企接口 |
| `CASH` | 现金 | 收款人 | 大额现金需主管审批；**供应商扣款收款(F07)固定使用此方式** |

| `BILL_DRAFT` | 银行承兑汇票 | 票号、到期日、出票行 | 走票据流程 |
| `BILL_COMMERCIAL` | 商业承兑汇票 | 票号、到期日、出票人 | 风险较高需审批 |

## 7. 非功能性需求

| 维度 | 要求 |
|:---|:---|
| **性能** | 单次核销响应 < 2秒; 列表查询 < 500ms |
| **并发** | 支持5个财务人员同时操作不冲突 |
| **数据一致性** | 核销操作必须在事务中完成; 收款金额+核销总额+预收余额 三者必须平衡 |
| **安全性** | 收款单审核后不可编辑; 反核销需要独立权限; 金额变更留痕 |
| **审计追踪** | 所有核销/反核销操作记录操作人、时间、IP、前后值 |
| **可用性** | 核销操作失败必须可重试; 不允许出现重复核销 |
