## 1. 模块定位与业务目标

### 1.1 一句话定义

**总账凭证引擎是ERP财务系统的核心枢纽，负责会计科目管理、凭证全生命周期管理、辅助核算、自动过账、期末结转与财务报表数据支撑，是所有业务模块最终写入的统一出口。**

### 1.3 核心业务目标

| 目标 | 指标 | 当前痛点 |
|:---|:---|:---|
| **凭证准确率** | 借贷平衡率 100%，辅助核算完整率 ≥99% | 手工做分录容易出错漏项 |
| **过账时效** | 业务审核通过 → 凭证过账 ≤ 1秒 | 手工入账延迟，月末突击 |
| **科目规范** | 统一科目体系，禁用未授权科目 | 科目代码混乱，各自为政 |
| **期末结账** | 月末结账从3天缩短到半天 | 手工结转耗时长、易遗漏 |
| **审计追溯** | 每笔凭证100%可追溯来源业务单据 | 缺乏业务凭证关联 |
| **多维度查询** | 按科目/辅助核算/期间多维度查余额 | 只能看科目总余额 |

### 1.4 用户角色

| 角色 | 权限范围 | 典型操作 |
|:---|:---|:---|
| **财务会计** | 创建/编辑草稿凭证 | 手工录入凭证、查看明细账 |
| **财务主管** | 审核所有凭证、反审核 | 凭证审核、科目管理 |
| **出纳** | 仅查看银行相关凭证 | 查询银行存款明细账 |
| **财务总监** | 反结账、科目变更审批 | 期末管理、科目调整 |
| **系统管理员** | 科目体系配置、凭证模板维护 | 参数配置、模板管理 |

### 3.2 表结构DDL

#### 表1: gl_period（会计期间表）

```sql
CREATE TABLE gl_period (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    period_year     INT          NOT NULL COMMENT '会计年度 2026',
    period_month    TINYINT      NOT NULL COMMENT '会计月份 1-12',
    period_name     VARCHAR(20)  NOT NULL COMMENT '期间名称 2026-04',
    
    -- 期间状态
    status          TINYINT      NOT NULL DEFAULT 0 
        COMMENT '0=未开启 1=已开账 2=已结账 3=已反结账',
    
    -- 日期范围
    begin_date      DATE         NOT NULL COMMENT '期间起始日',
    end_date        DATE         NOT NULL COMMENT '期间截止日',
    
    -- 结账信息
    closed_by       BIGINT       DEFAULT NULL COMMENT '结账人',
    closed_at       DATETIME     DEFAULT NULL COMMENT '结账时间',
    
    -- 附加
    remark          VARCHAR(200) DEFAULT NULL,
    created_at      DATETIME     DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    UNIQUE KEY uk_period (period_year, period_month)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='会计期间表';
```

## 4. 业务规则与凭证模板

### 4.1 凭证编号规则

```
格式: GL + YYYYMM + 6位序号
示例: GL202604000001 ~ GL202604999999

规则:
• GL = General Ledger 缩写
• 按月份重置序号
• 冲销凭证追加 -R 后缀: GL202604000005-R
• 删除后不填补空号
• 作废后原号作废，新生成新号
```

### 4.2 凭证字规则

| 凭证字 | 使用场景 | 典型分录 |
|:---:|:---|:---|
| **收** | 涉及现金/银行存款借方的凭证 | 收款、票据贴现、票据到期兑付 |
| **付** | 涉及现金/银行存款贷方的凭证 | 付款、票据到期付款、质押赎回 |
| **转** | 不涉及现金/银行存款的凭证 | 预收转应收、票据背书、损益结转 |

凭证字由系统根据分录中的科目自动判断：
```python
def auto_detect_voucher_word(entries: list) -> str:
    """根据分录自动判断凭证字"""
    has_cash_or_bank_debit = any(
        e['account_code'].startswith('1001') or e['account_code'].startswith('1002')
        and e['direction'] == 1  # 借方
        for e in entries
    )
    has_cash_or_bank_credit = any(
        e['account_code'].startswith('1001') or e['account_code'].startswith('1002')
        and e['direction'] == 2  # 贷方
        for e in entries
    )
    
    if has_cash_or_bank_debit and not has_cash_or_bank_credit:
        return '收'
    elif has_cash_or_bank_credit and not has_cash_or_bank_debit:
        return '付'
    else:
        return '转'
```

### 7.2 业务规则控制

| 规则 | 说明 |
|:---|:---|
| **制审分离** | 制单人 ≠ 审核人，系统强制校验 |
| **借贷平衡** | 过账前强制校验 ∑借方 = ∑贷方 (允许±0.01) |
| **期间控制** | 已结账期间不可新增/修改/过账凭证 |
| **科目状态** | 停用科目不可录入凭证分录 |

| **冲销控制** | 冲销凭证金额不超过原凭证，跨月冲销需总监审批 |
| **删除控制** | 仅草稿态可删除，已审核/过账不可删除 |
| **凭证编号** | 自动生成，不可修改，不可重复 |

def generate_summary(source_module: str, event_type: str, 
                     context: dict) -> str:
    """自动生成凭证摘要"""
    
    templates = {
        "CRM.RECEIPT.NORMAL":       "收{customer_name}{suffix}-SK{receipt_no}",
        "CRM.RECEIPT.PREPAY":       "收{customer_name}预付款-SK{receipt_no}",
        "CRM.RECEIPT.BILL":         "收{customer_name}承兑汇票-BILL{bill_no}",
        "PM.PAYMENT.NORMAL":        "付{supplier_name}{suffix}-FK{payment_no}",
        "PM.PAYMENT.PREPAY":        "预付{supplier_name}货款-FK{payment_no}",
        "PM.PAYMENT.BILL":          "付{supplier_name}承兑汇票-FK{payment_no}",
        "BM.BILL.RECEIVE":          "收{customer_name}票据-BILL{bill_no}",
        "BM.BILL.ENDORSE":          "背书{supplier_name}票据-BILL{bill_no}",
        "BM.BILL.DISCOUNT":         "{bill_no}票据贴现",
        "BM.BILL.CASHED":           "{bill_no}票据到期兑付",
        "PBM.BILL.ISSUE":           "开{supplier_name}应付票据-PBILL{bill_no}",
        "PBM.BILL.SETTLE":          "付{supplier_name}票据到期-PBILL{bill_no}",
        "GL.PERIOD.PROFIT_INCOME":  "结转本期损益-收入类",
        "GL.PERIOD.PROFIT_EXPENSE": "结转本期损益-费用类",
    }
    
    template = templates.get(f"{source_module}.{event_type}", "")
    return template.format(**context)
