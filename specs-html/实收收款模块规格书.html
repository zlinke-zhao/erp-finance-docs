1. 模块定位与业务目标

1.1 一句话定义

实收收款模块是ERP系统中"资金流入"的核心枢纽，负责从银行资金到账开始，经过收款登记、应收核销、凭证生成，最终写入总账的全流程管理。

1.2 在整体流程中的位置 

销售订单 → 送货单 → 应收对账 → 税务开票 → 发票录入 ─┐
                                                    │
采购申请 → 采购订单 → 收货验收 → 应付对账 ──────────┤
                                                    ▼
                                            ┌──────────────┐
                                            │ 🔄 实收收款   │ ◄── 本模块
                                            │ (本规格书)    │
                                            └──────┬───────┘
                                                   │
                                                   ▼
                                            ┌──────────────┐
                                            │ 🧾 总账凭证    │ ← P0待建
                                            │              │
                                            │ 借:银行存款   │
                                            │ 贷:应收账款   │
                                            └──────────────┘

1.3 核心业务目标

目标指标当前痛点
**收款可追溯**每笔收款100%关联到具体应收单据手工Excel对账，容易漏记
**核销准确率**自动核销匹配率≥90%，人工确认率≤10%全靠人眼逐笔核对
**账龄准确性**应收余额与实际回款实时同步月末才调一次，平时不准
**预收透明度**预收余额实时可见、自动抵扣预收款记在脑子里或散落各处
**凭证自动化**收款审核通过后自动生成凭证手工做分录，容易出错
**到账时效**从银行到账到系统登记≤4小时发现晚、登记不及时

1.4 用户角色

角色权限范围典型操作
**财务会计**创建/编辑/删除收款单录入收款、核销操作、查看报表
**财务主管**审核所有收款单、审批异常审核、红冲、权限审批
**出纳**仅录入和查询登记到账信息（不可核销）
**销售经理**只读查看自己客户的收款情况回款进度、客户欠款
**系统管理员**配置科目模板、核销规则、银行账户参数配置

2.2 功能详细说明

#### F01. 收款单管理（核心）

功能点描述优先级
**F01-01** 新增收款单支持手工录入、银企导入两种创建方式。必填：客户、金额、日期、收款方式、银行账户P0
**F01-02** 收款方式支持银行转账 / 现金 / POS刷卡 / 支付宝 / 微信支付 / 承兑汇票 / 其他P0
**F01-03** 多币种支持人民币及常用外币(USD/EUR/HKD等)，按汇率自动换算本位币P1
**F01-04** 草稿机制未审核前为草稿状态，可编辑；审核后锁定不可修改P0
**F01-05** 批量导入通过Excel模板批量导入多笔收款，支持校验和错误提示P1
**F01-06** 关联附件上传银行回单图片/电子回单PDF作为原始凭证P1

#### F02. 核销处理（核心中的核心）

功能点描述优先级
**F02-01** 待核销列表显示该客户所有未结清/部分核销的应收单据，含订单号、到期日、已收、未收P0
**F02-02** 自动核销当收款金额与某笔应收完全一致且仅有一笔待收时，一键自动核销P0
**F02-03** 半自动核销系统推荐最优核销方案（基于金额+账龄+历史记录），用户确认后执行P0
**F02-04** 手工核销用户手动选择一笔或多笔应收进行核销分配P0
**F02-05** 1对1核销单笔收款全额核销单笔应收P0
**F02-06** 多对1核销多笔收款累计核销同一笔大额应收（分期付款）P0
**F02-07** 1对多核销单笔大额收款分配给多笔应收（合并付款）P0
**F02-08** 多对多核销复杂场景：多笔收款分配给多笔应收，需明确指定每笔分配金额P1
**F02-09** 部分核销收款金额小于应收时，部分核销并标记剩余未收P0
**F02-10** 反核销（撤销）已核销的收款单允许反核销（需权限），恢复应收未收状态P1
**F02-11** 尾差自动处理≤¥1 或 ≤0.1%的差异自动归入财务费用-尾差调整科目P0
**F02-12** 折扣处理因提前付款享受折扣时，差额计入财务费用P1

#### F03. 预收款管理

功能点描述优先级
**F03-01** 预收款登记无对应订单的收款自动挂入客户预收账户P0
**F03-02** 预收余额查询每个客户的预收余额实时可见P0
**F03-03** 预收自动抵扣产生新应收时，若该客户有预收余额，提示是否使用预收核销P0
**F03-04** 预收退款超期未用或其他原因退回预收款P2
**F03-05** 预收明细台账预收的每一笔进出记录完整追溯P1

#### F04. 银企对接（Phase 2）

功能点描述优先级
**F04-01** 流水定时拉取通过银企接口每日凌晨自动拉取前一日的银行流水P2
**F04-02** 智能匹配算法基于对方户名+金额+摘要+历史的综合权重算法P2
**F04-03** 待认领池无法自动匹配的流水归集，定期提醒清理P2
**F04-04** 对账报告银行流水 vs 系统收款单的对账差异报告P2

#### F05. 凭证生成（依赖总账引擎）

功能点描述优先级
**F05-01** 凭证模板配置不同收款场景对应的借贷方科目模板（见§4）P0
**F05-02** 自动生成凭证收款审核通过后根据核销结果自动生成记账凭证P0
**F05-03** 凭证摘要规则自动生成规范的凭证摘要（含客户名、收款单号、关联订单）P1
**F05-04** 合并生成凭证同一客户同一天的多笔小金额收款合并为一张凭证P2

#### F06. 报表与分析

功能点描述优先级
**F06-01** 回款统计按客户/期间/销售员的回款金额、回款率排名P1
**F06-02** 应收账龄分析按账龄区间(30/60/90/180天+)分布，逾期预警P1
**F06-03** 现金流量表经营活动现金流入分类汇总P1
**F06-04** 收款日报/周报定时推送收款概况给管理层P2

3.2 数据库DDL

#### 表1: receipt（收款单主表） 

CREATE TABLE receipt (
    id                BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    
    -- === 基本信息 ===
    receipt_no        VARCHAR(20)  NOT NULL UNIQUE  COMMENT '收款单编号(自动生成)',
    customer_id       BIGINT       NOT NULL        COMMENT '客户ID',
    
    -- === 金额信息 ===
    amount            DECIMAL(14,2) NOT NULL       COMMENT '收款金额(原币)',
    currency          VARCHAR(6)   NOT NULL DEFAULT 'CNY' COMMENT '币种',
    exchange_rate     DECIMAL(12,6) DEFAULT 1.000000 COMMENT '汇率',
    amount_cny        DECIMAL(14,2) NOT NULL        COMMENT '折合人民币金额',
    unwriteoff_amount DECIMAL(14,2) NOT NULL        COMMENT '未核销余额(初始=amount,核销后递减)',
    
    -- === 收款信息 ===
    received_date     DATE         NOT NULL        COMMENT '收款日期',
    payment_method    VARCHAR(20)  NOT NULL        COMMENT '收款方式: BANK_TRANSFER/CASH/POS/ALIPAY/WECHAT/BILL/DRAFT/OTHER',
    bank_account_id   BIGINT       DEFAULT NULL     COMMENT '收款银行账户ID',
    bank_ref_no       VARCHAR(50)  DEFAULT NULL     COMMENT '银行参考号/流水号',
    
    -- === 来源信息 ===
    source_type       VARCHAR(20)  NOT NULL DEFAULT 'MANUAL' COMMENT '来源: MANUAL/BANK_IMPORT/BATCH/API',
    source_ref        VARCHAR(50)  DEFAULT NULL     COMMENT '来源引用(银行流水ID等)',
    
    -- === 关联信息 ===
    voucher_id        BIGINT       DEFAULT NULL     COMMENT '生成的凭证ID(关联总账模块)',
    
    -- === 附件与备注 ===
    attachment_url    VARCHAR(500) DEFAULT NULL     COMMENT '银行回单附件URL',
    remark            VARCHAR(500) DEFAULT NULL     COMMENT '备注',
    
    -- === 状态 ===
    status            VARCHAR(16)  NOT NULL DEFAULT 'DRAFT' COMMENT 'DRAFT/VERIFIED/CANCELLED',
    
    -- === 审计 ===
    created_by        BIGINT       DEFAULT NULL     COMMENT '创建人ID',
    created_at        DATETIME     DEFAULT CURRENT_TIMESTAMP,
    updated_by        BIGINT       DEFAULT NULL     COMMENT '最后修改人',
    updated_at        DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    verified_by       BIGINT       DEFAULT NULL     COMMENT '审核人',
    verified_at       DATETIME     DEFAULT NULL     COMMENT '审核时间',
    
    -- === 索引 ===
    INDEX idx_customer_date (customer_id, received_date),
    INDEX idx_status (status),
    INDEX idx_source (source_type, source_ref),
    INDEX idx_received_date (received_date)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 
COMMENT='收款单主表 - 记录每一笔实收款项';

#### 表2: receipt_writeoff（核销记录表） 

CREATE TABLE receipt_writeoff (
    id                BIGINT PRIMARY KEY AUTO_INCREMENT,
    
    -- === 关联 ===
    receipt_id        BIGINT       NOT NULL COMMENT '收款单ID',

    discount_amount   DECIMAL(14,2) DEFAULT 0      COMMENT '折扣/坏账金额(若有)',
    
    -- === 核销类型 ===
    writeoff_type     VARCHAR(16)  NOT NULL DEFAULT 'MANUAL' 
        COMMENT '核销类型: AUTO(全自动)/SEMI_AUTO(半自动)/MANUAL(手工)/PREPAY(预收抵扣)/REVERSE(反核销)',
    
    -- === 尾差处理 ===
    rounding_diff     DECIMAL(8,2)  DEFAULT 0      COMMENT '尾差金额(若有)',
    rounding_acct     VARCHAR(30)  DEFAULT NULL     COMMENT '尾差计入科目',
    
    -- === 时间戳 ===
    writeoff_date     DATE         NOT NULL        COMMENT '核销日期',
    created_by        BIGINT       NOT NULL        COMMENT '核销操作人',
    created_at        DATETIME     DEFAULT CURRENT_TIMESTAMP,
    
    -- === 索引(关键!) ===
    UNIQUE INDEX uk_receipt_ar (receipt_id, ar_invoice_id),
    INDEX idx_ar (ar_invoice_id),
    INDEX idx_so (so_ref_id),
    INDEX idx_writeoff_date (writeoff_date)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 
COMMENT='收款核销记录 - 每次核销操作一条记录,支持多次部分核销';

#### 表3: customer_prepay（客户预收账户表） 

CREATE TABLE customer_prepay (
    id                BIGINT PRIMARY KEY AUTO_INCREMENT,
    customer_id       BIGINT       NOT NULL UNIQUE  COMMENT '客户ID',
    
    -- === 预收余额 ===
    current_balance   DECIMAL(14,2) NOT NULL DEFAULT 0 COMMENT '当前可用预收余额',
    frozen_balance    DECIMAL(14,2) NOT NULL DEFAULT 0 COMMENT '冻结余额(已关联但未最终核销)',
    total_in          DECIMAL(14,2) NOT NULL DEFAULT 0 COMMENT '累计预收入',
    total_out         DECIMAL(14,2) NOT NULL DEFAULT 0 COMMENT '累计预收使用/退还',
    
    -- === 更新时间 ===
    last_updated_at   DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    INDEX idx_customer (customer_id)

#### 表4: prepay_transaction（预收流水明细表） 

CREATE TABLE prepay_transaction (
    id                BIGINT PRIMARY KEY AUTO_INCREMENT,
    customer_id       BIGINT       NOT NULL        COMMENT '客户ID',
    
    -- === 变动信息 ===
    trans_type        VARCHAR(16)  NOT NULL        COMMENT 'IN(收到预收)/OUT(预收核销)/REFUND(退款)/ADJUST(调整)',
    amount            DECIMAL(14,2) NOT NULL       COMMENT '变动金额(正数表示增加预收,负数表示减少)',
    balance_after     DECIMAL(14,2) NOT NULL        COMMENT '变动后余额',
    
    -- === 关联来源 ===
    ref_type          VARCHAR(20)  NOT NULL        COMMENT '关联类型: RECEIPT/AR_INVOICE/REFUND/MANUAL',
    ref_id            BIGINT       NOT NULL        COMMENT '关联单据ID',
    remark            VARCHAR(200) DEFAULT NULL     COMMENT '备注',
    
    created_at        DATETIME     DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_customer (customer_id),
    INDEX idx_trans_type (trans_type)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 
COMMENT='预收变动流水 - 每一笔预收的增加/减少都有记录,可审计追溯';

4. 业务规则与凭证模板

4.1 六种收款场景的会计分录

场景代码场景描述借方贷方备注
**S01**正常收款核销`1002 银行存款-xx银行``1122 应收账款-XX客户`最常见的标准场景
**S02**纯预收款（无订单）`1002 银行存款-xx银行``2203 预收账款-XX客户`先款后货/VIP存入
**S03**预收转应收核销`2203 预收账款-XX客户``1122 应收账款-XX客户`新订单产生时的内部转账
**S04**收款+提前付款折扣`1002 银行存款` + `6603 财务费用-折扣支出``1122 应收账款`客户享早鸟折扣
**S05**收到承兑汇票`1121 应收票据-承兑汇票``1122 应收账款-XX客户`以票据代替现金
**S06**汇票贴现/兑现`1002 银行存款` + `6603 财务费用-贴现息``1121 应收票据-承兑汇票`票据到期或贴现

4.3 收款单编号规则 

格式: SK + 年月 + 5位序号
示例: SK20260400001 ~ SK20260499999

规则:
• SK = ShouKou(收款)缩写
• 按月份重置序号
• 删除后不填补空号(保持连续性)
• 作废后原号作废,新生成新号

4.4 收款方式枚举

code名称需填字段特殊处理
`BANK_TRANSFER`银行转账银行账户、流水号可对接银企接口
`CASH`现金收款人大额现金需主管审批
`POS`POS刷卡终端号、卡号后四位含手续费扣减
`ALIPAY`支付宝交易号可对接API
`WECHAT`微信支付交易号可对接API
`BILL_DRAFT`银行承兑汇票票号、到期日、出票行走票据流程
`BILL_COMMERCIAL`商业承兑汇票票号、到期日、出票人风险较高需审批
`OTHER`其他说明需填写备注

7. 非功能性需求

维度要求
**性能**单次核销响应 < 2秒; 列表查询 < 500ms
**并发**支持5个财务人员同时操作不冲突
**数据一致性**核销操作必须在事务中完成; 收款金额+核销总额+预收余额 三者必须平衡
**安全性**收款单审核后不可编辑; 反核销需要独立权限; 金额变更留痕
**审计追踪**所有核销/反核销操作记录操作人、时间、IP、前后值
**可用性**核销操作失败必须可重试; 不允许出现重复核销

8. 测试用例要点

用例类别测试点预期结果
**正常流程**1对1全额核销应收余额清零,收款单状态=已核销
**正常流程**部分核销(收款<应收)应收减少,剩余未收>0
**边界测试**尾差¥0.05自动归入财务费用
**边界测试**核销金额恰好等于应收余额精确为0
**异常测试**重复核销同一张应收系统拦截,提示"已核销"
**异常测试**核销金额超过未收余额校验拦截,"核销金额超限"
**异常测试**已审核收款单编辑权限拦截,"已审核单据不可修改"
**预收场景**有预收时新建应收弹窗提示"该客户有预收¥XXX,是否使用?"
**预收场景**预收不足抵扣全部应收部分抵扣+部分正常核销
