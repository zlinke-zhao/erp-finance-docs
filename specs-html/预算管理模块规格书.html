    • 一、模块概述

    预算管理是企业财务管控的核心工具，通过"编-审-控-调-分析"闭环，实现资金的事前计划、事中控制、事后分析。

    1.1 核心价值

    价值点说明
    资金可视化事前规划资金使用，避免资金黑洞
    过程管控事中控制预算执行，超限预警/阻断
    绩效考核部门/项目维度预算执行率考核
    决策支撑实时预算执行分析，支撑经营决策

    1.2 对标金蝶云星空

    金蝶云星空预算管理模块功能：

  • **预算编制**：支持年度/季度/月度多周期预算编制
  • **预算审批**：多级审批流程，支持预算冻结/解冻
  • **预算控制**：严格控制/弹性控制/不控制三种模式
  • **预算调整**：支持预算调整申请与审批
  • **预算分析**：执行率分析、预算与实际对比、滚动预测
  • **预算考核**：部门/项目维度预算执行率考核

    • 三、详细功能说明

    3.1 预算基础设置

    3.1.1 预算周期定义

    字段说明示例
    周期类型年度/季度/月度月度
    开始月份预算周期起始月1月
    结束月份预算周期结束月12月
    编制提前期提前多少天开始编制30天
    锁定规则何时锁定不可修改审批通过后

    3.1.2 预算组织架构

    CODE
    组织层级：
├── 公司级预算
├── 事业部/部门级预算
│   ├── 销售部
│   ├── 采购部
│   ├── 生产部
│   └── 管理部门
└── 项目级预算
    ├── 项目A
    └── 项目B

    3.1.3 预算科目体系

    科目类型方向说明
    收入类预算增加销售收入、其他收入
    成本类预算减少直接成本、制造费用
    费用类预算减少管理费用、销售费用
    资产类预算变动固定资产采购、在建工程

    3.2 预算编制

    3.2.1 年度预算编制流程

    CODE
    开始 -> 选择预算年度 -> 选择编制部门 -> 选择预算模板
  -> 按模板填报数据 -> 数据校验 -> 提交审批 -> 结束

    3.2.2 预算编制规则

    规则类型说明配置方式
    零基预算从零开始编制每年度重置
    增量预算基于上年调整增量比例配置
    弹性预算基于业务量调整公式定义

    3.2.3 预算数据来源

    来源说明
    手工录入直接在系统填写
    Excel导入模板下载-填写-导入
    历史数据基于上年实际/预算复制
    业务计划关联销售计划、生产计划

    3.3 预算审批

    3.3.1 审批流程

    CODE
    编制人提交 -> 部门主管审批 -> 财务经理审批 -> 财务总监审批 -> 发布
     ↓              ↓              ↓
   驳回          驳回          驳回
     ↓              ↓              ↓
  返回编制      返回编制      返回编制

    3.3.2 审批权限矩阵

    预算金额审批层级审批人
    < 10万1级部门主管
    10-50万2级部门主管 + 财务经理
    50-100万3级部门主管 + 财务经理 + 财务总监
    >= 100万4级+ 总经理

    3.4 预算控制

    3.4.1 控制模式

    控制模式说明触发动作
    严格控制不得超预算直接拒绝报销
    弹性控制可超一定比例预警 + 审批升级
    不控制仅记录分析仅记录，无拦截

    3.4.2 控制维度

    CODE
    预算控制维度：
├── 维度1：部门维度
│   └── 各部门独立控制
├── 维度2：项目维度
│   └── 各项目独立控制
├── 维度3：费用项维度
│   └── 各费用项独立控制
└── 维度4：时间维度
    └── 月度/季度/年度控制

    3.4.3 预警规则

    预警级别触发条件通知方式
    L1 提醒达到 70%站内信
    L2 预警达到 90%站内信 + 邮件
    L3 警告达到 100%全渠道通知
    L4 超限超过 100%全渠道 + 阻断审批

    3.5 预算调整

    3.5.1 调整类型

    调整类型说明
    科目间调整A费用项调减，B费用项调增
    部门间调整A部门调减，B部门调增
    时间维度调整Q1调减，Q2调增
    追加预算整体追加，需额外审批

    3.5.2 调整流程

    CODE
    调整申请 -> 填写调整金额 -> 选择调整原因 -> 提交审批
  -> 财务经理审批 -> 财务总监审批（>50万）-> 执行调整

    3.6 预算分析

    3.6.1 分析报表

    报表名称说明
    预算执行进度表各部门/项目预算执行率
    预算与实际对比表预算vs实际vs差异
    预算差异分析表差异原因分析
    滚动预测表基于执行数据的未来预测
    预算考核表各部门考核结果

    3.6.2 分析维度

    CODE
    分析维度：
├── 时间维度：年度/季度/月度
├── 组织维度：公司/事业部/部门/项目
├── 科目维度：收入/成本/费用
└── 对比维度：预算/实际/差异/执行率

    • 四、数据库设计

    4.1 核心表结构

    预算主表（BUDGET_MASTER）

    字段类型说明
    BUDGET_IDVARCHAR(32)预算ID（PK）
    BUDGET_YEARINT预算年度
    BUDGET_PERIODVARCHAR(10)预算周期（年度/季度/月度）
    ORG_IDVARCHAR(32)组织ID
    DEPT_IDVARCHAR(32)部门ID
    PROJECT_IDVARCHAR(32)项目ID（可选）
    BUDGET_AMOUNTDECIMAL(18,2)预算金额
    STATUSVARCHAR(20)状态（草稿/审批中/已发布/已锁定）
    CREATED_BYVARCHAR(32)创建人
    CREATED_TIMEDATETIME创建时间
    UPDATED_BYVARCHAR(32)更新人
    UPDATED_TIMEDATETIME更新时间

    预算明细表（BUDGET_DETAIL）

    字段类型说明
    DETAIL_IDVARCHAR(32)明细ID（PK）
    BUDGET_IDVARCHAR(32)预算ID（FK）
    ACCOUNT_IDVARCHAR(32)科目ID
    JAN_AMOUNTDECIMAL(18,2)1月金额
    FEB_AMOUNTDECIMAL(18,2)2月金额
    .........
    DEC_AMOUNTDECIMAL(18,2)12月金额
    ANNUAL_AMOUNTDECIMAL(18,2)年度合计

    预算调整表（BUDGET_ADJUST）

    字段类型说明
    ADJUST_IDVARCHAR(32)调整ID（PK）
    BUDGET_IDVARCHAR(32)原预算ID
    ADJUST_TYPEVARCHAR(20)调整类型
    FROM_ACCOUNTVARCHAR(32)调出科目
    TO_ACCOUNTVARCHAR(32)调入科目
    ADJUST_AMOUNTDECIMAL(18,2)调整金额
    ADJUST_REASONVARCHAR(500)调整原因
    STATUSVARCHAR(20)状态
    APPROVAL_IDVARCHAR(32)审批ID

    预算执行表（BUDGET_EXECUTION）

    字段类型说明
    EXEC_IDVARCHAR(32)执行ID（PK）
    BUDGET_IDVARCHAR(32)预算ID
    BUDGET_DETAIL_IDVARCHAR(32)预算明细ID
    DOCUMENT_TYPEVARCHAR(20)单据类型（报销/采购等）
    DOCUMENT_IDVARCHAR(32)单据ID
    EXEC_AMOUNTDECIMAL(18,2)执行金额
    EXEC_TIMEDATETIME执行时间

    • 五、业务流程图

    详见配套流程图文件：预算管理总流程图.html

    5.1 核心流程

    CODE
    1. 预算编制流程
   启动编制 -> 模板选择 -> 数据填报 -> 校验 -> 提交审批 -> 发布

2. 预算控制流程
   报销申请 -> 预算检查 -> 额度判断 -> 通过/预警/拒绝 -> 执行/调整

3. 预算调整流程
   调整申请 -> 原因填写 -> 审批 -> 执行调整 -> 更新台账

4. 预算分析流程
   数据采集 -> 执行率计算 -> 差异分析 -> 生成报表 -> 考核评价

    • 八、实施建议

    8.1 实施步骤

    CODE
    Phase 1: 基础设置（1-2周）
  - 组织架构定义
  - 科目体系定义
  - 预算模板配置

Phase 2: 编制上线（2-3周）
  - 历史数据迁移
  - 用户培训
  - 试运行

Phase 3: 控制启用（1-2周）
  - 控制规则配置
  - 预警规则配置
  - 正式启用

Phase 4: 分析深化（持续）
  - 分析模型优化
  - 考核体系完善

    8.2 关键成功因素

    因素说明
    领导支持高层支持是预算管理成功的关键
    数据质量基础数据准确是预算有效的前提
    持续优化根据执行情况持续调整预算模型
    全员参与各部门参与才能保证预算落地
