1. 模块概述

1.1 业务背景

公司业务覆盖东南亚及中东4个国家，需处理海外进项（供应商开给我们的发票）和销项（我们开给客户的发票）两类场景。海外发票与国内增值税发票存在本质差异：无统一税局API验真、各国税制独立、多币种处理、"票随货走"等特殊业务模式。

1.2 模块定位

本模块是 发票查验模块规格书.md 的海外扩展，不独立存在。核心原则： `` 一个发票录入入口 → invoice_region 字段分流 → 国内走API验真 / 海外走人工审核 ↓ 审核通过后 → 应收/应付挂账 + 凭证生成（完全复用） `

1.3 覆盖国家

9. 风险控制

9.1 审核风控规则

| 规则 | 触发条件 | 处理方式 | |:---|:---|:---| | 税号格式校验 | 税号不符合该国格式 | 前端拦截，不允许提交 | | 金额合理性 | 单张发票折算CNY > 100万 | 自动升级审批（需财务主管） | | 同一供应商频率 | 同一供应商一周内提交 > 10张 | 标记为高风险，需主管复核 | | 汇率偏离预警 | 手动修改汇率偏离银行中间价 > 3% | 系统警告，需备注原因 | | 附件缺失 | 提交时未上传附件 | 前端拦截，附件为必填 | | 重复发票检查 | 同一海外发票编号已存在 | 前端警告，允许关联但不允许重复挂账 |

10. 验真API预留设计

10.1 接口抽象层

`python class OverseasInvoiceVerifier(ABC): """海外发票验真抽象基类""" @abstractmethod def verify(self, invoice_data: dict) -> VerifyResult: """验真入口""" pass @abstractmethod def is_available(self) -> bool: """该渠道是否可用""" pass class ZATCAVerifier(OverseasInvoiceVerifier): """沙特 ZATCA 验真 — 未来对接""" # ZATCA FATOORAH 验证接口 # 需要沙特方的API密钥和证书 pass class GDTVerifier(OverseasInvoiceVerifier): """越南 GDT 验真 — 未来对接""" # 越南电子发票平台API pass class ManualVerifier(OverseasInvoiceVerifier): """人工审核 — 当前默认实现""" def verify(self, invoice_data): return VerifyResult( status="PENDING_MANUAL", message="等待人工审核" ) def is_available(self): return True `

10.2 验真策略路由

`python class VerificationRouter: """根据国家自动选择验真渠道""" STRATEGY = { "CN": "tencent_cloud", # 腾讯云API "SA": "zatca", # 未来: ZATCA "VN": "gdt", # 未来: GDT "TH": "manual", # 当前: 人工 "PH": "manual", # 当前: 人工 } def get_verifier(self, region: str) -> OverseasInvoiceVerifier: strategy = self.STRATEGY.get(region, "manual") # 根据策略返回对应的验真器 # 当前阶段: SA/VN/TH/PH 都返回 ManualVerifier pass ` ---
