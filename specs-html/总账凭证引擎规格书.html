1. 模块定位与业务目标

1.1 一句话定义

总账凭证引擎是ERP财务系统的核心枢纽，负责会计科目管理、凭证全生命周期管理、辅助核算、自动过账、期末结转与财务报表数据支撑，是所有业务模块最终写入的统一出口。

1.2 在整体架构中的位置

`` ┌──────────────┐ ┌──────────────┐ ┌──────────────┐ ┌──────────────┐ │ 发票查验 │ │ 实收收款 │ │ 实付付款 │ │ 票据往来 │ │ (IV) │ │ (CRM) │ │ (PM) │ │ (BM) │ └──────┬───────┘ └──────┬───────┘ └──────┬───────┘ └──────┬───────┘ │ │ │ │ │ S01~S06 │ P01~P08 │ B01~B08 │ PA01~PA07 └────────────────┴────────────────┴────────────────┘ │ ┌───────┴───────┐ │ 🧾 总账凭证 │ ◄── 本模块 (P0) │ 引擎(GLE) │ └───────┬───────┘ │ ┌───────────────┼───────────────┐ ▼ ▼ ▼ ┌───────────┐ ┌───────────┐ ┌───────────┐ │ 资产负债表 │ │ 利润表 │ │ 现金流量表 │ │ (BS) │ │ (PL) │ │ (CF) │ └───────────┘ └───────────┘ └───────────┘ `

1.3 核心业务目标

| 目标 | 指标 | 当前痛点 | |:---|:---|:---| | 凭证准确率 | 借贷平衡率 100%，辅助核算完整率 ≥99% | 手工做分录容易出错漏项 | | 过账时效 | 业务审核通过 → 凭证过账 ≤ 1秒 | 手工入账延迟，月末突击 | | 科目规范 | 统一科目体系，禁用未授权科目 | 科目代码混乱，各自为政 | | 期末结账 | 月末结账从3天缩短到半天 | 手工结转耗时长、易遗漏 | | 审计追溯 | 每笔凭证100%可追溯来源业务单据 | 缺乏业务凭证关联 | | 多维度查询 | 按科目/辅助核算/期间多维度查余额 | 只能看科目总余额 |

4. 业务规则与凭证模板

4.1 凭证编号规则

` 格式: GL + YYYYMM + 6位序号 示例: GL202604000001 ~ GL202604999999 规则: • GL = General Ledger 缩写 • 按月份重置序号 • 冲销凭证追加 -R 后缀: GL202604000005-R • 删除后不填补空号 • 作废后原号作废，新生成新号 `

4.2 凭证字规则

| 凭证字 | 使用场景 | 典型分录 | |:---:|:---|:---| | 收 | 涉及现金/银行存款借方的凭证 | 收款、票据贴现、票据到期兑付 | | 付 | 涉及现金/银行存款贷方的凭证 | 付款、票据到期付款、质押赎回 | | 转 | 不涉及现金/银行存款的凭证 | 预收转应收、票据背书、损益结转 | 凭证字由系统根据分录中的科目自动判断： `python def auto_detect_voucher_word(entries: list) -> str: """根据分录自动判断凭证字""" has_cash_or_bank_debit = any( e['account_code'].startswith('1001') or e['account_code'].startswith('1002') and e['direction'] == 1 # 借方 for e in entries ) has_cash_or_bank_credit = any( e['account_code'].startswith('1001') or e['account_code'].startswith('1002') and e['direction'] == 2 # 贷方 for e in entries ) if has_cash_or_bank_debit and not has_cash_or_bank_credit: return '收' elif has_cash_or_bank_credit and not has_cash_or_bank_debit: return '付' else: return '转' `
