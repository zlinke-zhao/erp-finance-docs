一、模块定位与业务背景

1.1 模块地位

会计科目是财务管理系统的基础数据源，所有财务业务（凭证、账簿、报表、应收应付、固定资产、成本核算、预算管理）均依赖科目体系。一旦科目结构确定，后续模块的账务处理逻辑将被锚定。因此，会计科目设定是整个财务管理系统第一个需要完整规划的核心模块。

> 本模块对应金蝶云星空 → 基础资料 → 会计科目，支持多账套、多币别、数量核算、辅助核算等完整特性。

1.2 当前差距分析

基于 ERP 功能菜单（877 条）梳理结果，现有财务相关功能约 90 条，会计科目模块严重缺失，主要表现：

缺失项现状影响
会计科目体系❌ 无科目定义功能无法进行标准账务处理
科目级次/维度管理❌ 无无法支持多维度辅助核算
科目余额初始化❌ 无无法启用账务系统
科目受控字段❌ 无各模块字段标准不统一
多账套科��管理❌ 仅 1 个"公司帐套"无法支撑多组织财务集中

二、功能范围与边界

2.1 核心功能范围（Must Have）

功能域功能点说明
**科目体系管理**新增/编辑/删除/复制科目支持多级科目（1-10 级）
科目编码规则配置4-2-2-2-2 等自由定义
科目类型定义资产/负债/权益/成本/损益/表外
余额方向设置借/贷
现金/银行/数量科目标记影响现金流量表和成本核算
**辅助核算维度**部门/人员/供应商/客户/项目/物料/自定义维度支持多辅助核算并行
核算维度层级管理如部门树形结构
**账套级科目分配**科目与账套关联同一科目在不同账套可有不同控制策略
科目受控模块设置科目在哪些模块可用
**启用控制**科目启用/禁用禁用后不可新增凭证
已使用科目锁定有发生额的科目禁止删除
**余额初始化**科目期初余额录入支持多期间、多币别、数量
试算平衡校验借方合计=贷方合计

2.2 扩展功能范围（Should Have）

功能点说明
科目受控字段统一管理字段长度/必填/默认值
科目模板导入/导出Excel 模板批量新增
历史科目版本管理科目变更留痕
科目体系复制以现有账套科目为模板新建账套
多语言科目名称支持中英文双语

2.3 非功能范围（Won't Have）

  • 跨年结账/反结账：属于总账凭证管理模块
  • 凭证自动生成规则：属于凭证生成模块
  • 报表取数公式：属于财务报表模块
  • 与税务系统对接的科目映射：属于发票管理模块

三、用户故事（User Stories）

3.1 基础科目管理 

【故事 1】作为财务管理员，我需要新增末级科目，以便完成科目体系的搭建。
  - 给定：我已登录系统并进入"会计科目设定"界面
  - 当：我点击"新增科目"，填写编码"1002"、名称"银行存款"，选择余额方向为"借"、科目类型为"资产"
  - 则：系统保存科目，并在科目树中显示，刷新后可见

【故事 2】作为财务管理员，我需要在科目下新增下级科目，以便细化核算颗粒度。
  - 给定：科目"1002 银行存款"已存在
  - 当：我新增下级科目"100201 中国银行"
  - 则：原科目"1002"自动变为非末级（父级），不能再作为凭证借贷方使用

【故事 3】作为财务管理员，我需要复制科目，以便快速批量创建相似科目。
  - 给定：科目"6602 销售费用"及其辅助核算设置已配置完毕
  - 当：我选中该科目，点击"复制"，修改编码为"6603"、名称为"管理费用"
  - 则：新建科目继承原科目的余额方向、辅助核算等属性，仅编码和名称不同

【故事 4】作为财务管理员，我需要禁用已停用的科目，以便在界面上隐藏不再使用的科目。
  - 给定：科目"100199 其他货币资金"无任何发生额和余额
  - 当：我将该科目状态改为"禁用"
  - 则：该科目不再出现在凭证录入的科目选择列表中，但数据库记录保留

【故事 5】作为系统管理员，我需要删除错误录入的科目，以便修正初始化失误。
  - 给定：某科目从未被任何单据/凭证引用
  - 当：我选中该科目并点击"删除"
  - 则：系统弹出确认框，提示"确定删除？"，确认后删除并刷新列表

【故事 6】作为系统管理员，我需要防止删除已被使用的科目，以避免账务数据孤岛。
  - 给定：科目"1001 库存现金"已被某张凭证引用
  - 当：我尝试删除该科目
  - 则：系统拦截并提示"该科目已被使用，无法删除"

3.2 辅助核算维度管理 

【故事 7】作为财务管理员，我需要为科目绑定"部门"辅助核算，以便按部门归集���用。
  - 给定：科目"6602 销售费用"未设置辅助核算
  - 当：我勾选"部门"辅助核算并选择"多选"
  - 则：该科目在凭证录入时必须选择具体部门，系统按部门维度归集账簿

【故事 8】作为财务管理员，我需要为科目绑定"项目"辅助核算，以便按项目单独核算成本。
  - 给定：科目"5001 项目成本"未设置辅助核算
  - 当：我勾选"项目"辅助核算
  - 则：该科目在凭证录入时必须选择具体项目，可按项目出具辅助账簿

【故事 9】作为财务管理员，我需要科目同时支持多个辅助核算维度，以便满足复杂业务场景。
  - 给定：科目"660201 差旅费"需要同时按部门和项目核算
  - 当：我勾选"部门"和"项目"两个辅助核算维度
  - 则：该科目在凭证录入时需同时选择部门和项目，系统支持双向交叉账簿

3.3 账套与科目分配 

【故事 10】作为财务管理员，我需要将科目分配到特定账套，以便实现多账套独立核算。
  - 给定：系统中有"法人账套A"和"法人账套B"两个账套
  - 当：我将科目"1002 银行存款"分配给"账套A"
  - 则：该科目仅在账套A的凭证录入中可见，账套B不受影响

【故事 11】作为财务管理员，我需要设置科目在哪些模块受控，以便统一字段标准。
  - 给定：科目"1122 应收账款"需要在应收模块和总账模块受控
  - 当：我勾选"应收模块"和"总账模块"
  - 则：该科目的字段长度、必填等规则在两个模块保持一致

【故事 12】作为系统管理员，我需要复制整个账套的科目体系到新账套，以便快速启用新组织。
  - 给定：账套A有完整的科目体系（100+科目）
  - 当：我使用"以账套A为模板新建账套B"
  - 则：账套B自动复制账套A的所有科目及其属性，管理员仅需少量调整

3.4 余额初始化 

【故事 13】作为财务主管，我需要在系统启用前录入科目期初余额，以便财务数据衔接。
  - 给定：系统处于初始化期间，尚未开账
  - 当：我录入"1001 库存现金"的期初余额为借方 50,000 元
  - 则：系统保存期初余额，在试算平衡表中显示

【故事 14】作为财务主管，我需要系统自动校验试算平衡，以便确保初始化数据正确。
  - 给定：我已录入所有科目的期初余额
  - 当：我点击"试算平衡检查"
  - 则：系统计算所有资产类科目余额合计与所有负债+权益类科目余额合计
        - 若相等：提示"试算平衡，可以通过"
        - 若不相等：提示差异金额，列出疑似错误科目

【故事 15】作为财务主管，我需要录入多币别的期初余额，以便支持外币核算。
  - 给定：科目"1002 银行存款"已勾选"外币核算"
  - 当：我录入期初余额为本位币 100,000、美元 13,800（汇率 7.25）
  - 则：系统同时保存本位币余额和外币余额，汇率和币种一并记录

四、详细功能规格

4.1 科目体系结构

#### 4.1.1 科目编码规则

  • 编码格式：支持自由定义段数（1-5段）和每段长度（1-10位）
  • 推荐格式：4-2-2-2-2（共12位，5级）

- 示例：1001 → 1001.01 → 1001.01.01 → 1001.01.01.01

  • 编码约束：

- 同一级次下编码必须唯一

- 编码必须连续（不允许跳跃，如先录1001再录1003跳号）

- 下级科目编码必须以上级为前缀

#### 4.1.2 科目属性字段

字段名字段类型必填说明示例
科目IDGUID是主键`A1B2C3D4-E5F6-...`
科目编码VARCHAR(30)是唯一索引`1002.01.01`
科目名称NVARCHAR(100)是显示名称`中国银行深圳支行`
英文名称NVARCHAR(100)否国际化用`Bank of China SZ Branch`
科目类型ENUM是6种之一资产/负债/权益/成本/损益/表外
余额方向ENUM是借/贷借
末级标志BIT是自动计算1=末级，0=父级
数量核算BIT否是否启用数量辅助0/1
外币核算BIT否是否启用外币核算0/1
现金科目BIT否影响现金流量表0/1
银行科目BIT否影响银行日记账0/1
默认税率DECIMAL(5,2)否增值税默认税率13%
受控模块VARCHAR(200)否模块代码逗号分隔`AR,AP,GL`
启用状态BIT是是否启用1/0
排序号INT是同级排序1
账套IDGUID是所属账套引用 BD_AccountBook
创建时间DATETIME是自动`2026-04-08 12:00:00`
创建人VARCHAR(50)是自动`admin`
修改时间DATETIME否自动`2026-04-08 12:00:00`
修改人VARCHAR(50)否自动`admin`

#### 4.1.3 科目类型业务规则

类型余额方向典型科目辅助核算建议
资产借库存现金、银行存款、应收账款部门+供应商+项目
负债贷应付账款、短期借款、应付票据供应商+项目
权益贷实收资本、资本公积、未分配利润部门
成本借/贷生产成本、制造费用部门+项目+物料
损益借/贷主营业务收入、管理费用部门+项目
表外借/贷代管商品、受托加工物资自定义

4.2 辅助核算维度

#### 4.2.1 内置维度

维度代码维度名称是否树形说明
DEPT部门是引用组织架构模块
PERSON人员是引用人员档案
SUPPLIER供应商否引用供应商档案
CUSTOMER客户否引用客户档案
PROJECT项目是引用项目档案
MATERIAL物料否引用物料档案
BANK银行账户否引用银行账户档案
COSTITEM费用项目否自定义费用项
SELFDEF01~10自定义维度1~10可配置支持用户自定义

#### 4.2.2 辅助核算设置

  • 多选模式：每个科目同一维度可选"单选"或"多选"
  • 必填控制：科目在该维度可设为"必填"或"选填"
  • 组合控制：同一科目可绑定多个维度，维度间为"AND"关系
  • 维度层级：如"部门"支持到最末级部门为止

    LevelNo          INT DEFAULT 1,                  -- 科目级次（1-10）
    IsQuantityEnabled    BIT DEFAULT 0,             -- 数量核算
    IsForeignCurrencyEnabled BIT DEFAULT 0,        -- 外币核算
    IsCashAccount    BIT DEFAULT 0,                  -- 现金科目
    IsBankAccount    BIT DEFAULT 0,                  -- 银行科目
    DefaultTaxRate   DECIMAL(5,2) NULL,             -- 默认税率
    ControlledModules NVARCHAR(200) NULL,            -- 受控模块，逗号分隔
    IsEnabled        BIT DEFAULT 1,                  -- 启用状态
    SortOrder        INT DEFAULT 0,                  -- 同级排序
    CreatedDate      DATETIME DEFAULT GETDATE(),
    CreatedBy        NVARCHAR(50),
    ModifiedDate     DATETIME,
    ModifiedBy       NVARCHAR(50),
    -- 约束
    CONSTRAINT CK_SubjectType CHECK (SubjectType IN (N'Asset',N'Liability',N'Equity',N'Cost',N'ProfitLoss',N'OffBalance')),
    CONSTRAINT CK_BalanceDir CHECK (BalanceDirection IN (N'借',N'贷')),
    CONSTRAINT CK_LevelNo CHECK (LevelNo BETWEEN 1 AND 10),
    -- 唯一索引：同一账套下编码唯一
    CONSTRAINT UX_SubjectCode_PerBook UNIQUE (AccountBookID, SubjectCode),
    CONSTRAINT FK_Subject_AccountBook FOREIGN KEY (AccountBookID) REFERENCES dbo.BD_AccountBook(AccountBookID),
    CONSTRAINT FK_Subject_Parent FOREIGN KEY (ParentID) REFERENCES dbo.FA_AccountSubject(SubjectID)
);

4.4 服务端 API 设计（RESTful）

#### 4.4.1 科目管理 API

方法路径说明
`GET``/api/v1/account-books/{bookId}/subjects`分页查询科目列表（支持树形/列表切换）
`GET``/api/v1/account-books/{bookId}/subjects/tree`获取科目树（完整层级结构）
`GET``/api/v1/account-books/{bookId}/subjects/{id}`获取单个科目详情（含辅助核算）
`POST``/api/v1/account-books/{bookId}/subjects`新增科目
`PUT``/api/v1/account-books/{bookId}/subjects/{id}`修改科目
`DELETE``/api/v1/account-books/{bookId}/subjects/{id}`删除科目
`POST``/api/v1/account-books/{bookId}/subjects/{id}/copy`复制科目
`POST``/api/v1/account-books/{bookId}/subjects/{id}/enable`启用科目
`POST``/api/v1/account-books/{bookId}/subjects/{id}/disable`禁用科目
`POST``/api/v1/account-books/{bookId}/subjects/batch-import`批量导入（Excel）
`GET``/api/v1/account-books/{bookId}/subjects/export`导出 Excel

#### 4.4.2 辅助核算 API

方法路径说明
`GET``/api/v1/aux-dimensions`查询所有辅助核算维度定义
`GET``/api/v1/subjects/{subjectId}/aux-dimensions`获取科目绑定的辅助核算维度
`POST``/api/v1/subjects/{subjectId}/aux-dimensions`绑定辅助核算维度
`DELETE``/api/v1/subjects/{subjectId}/aux-dimensions/{dimCode}`解除辅助核算绑定

#### 4.4.3 期初余额 API

方法路径说明
`GET``/api/v1/account-books/{bookId}/init-balances`获取期初余额列表
`POST``/api/v1/account-books/{bookId}/init-balances`批量保存期初余额
`POST``/api/v1/account-books/{bookId}/init-balances/trial-balance`试算平衡校验
`POST``/api/v1/account-books/{bookId}/init-balances/lock`锁定期初余额（开账）

与现有 ERP 模块的数据对接现有业务单据可能引用科目制定过渡期方案：先并行，待新科目确认后批量迁移
