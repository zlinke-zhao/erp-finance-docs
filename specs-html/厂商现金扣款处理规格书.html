一、业务背景

1.1 问题描述

在与越南、泰国等海外供应商对账时，经常会出现品质扣款、迟延罚款等情况。传统做法是从应付账款中直接扣减，但这些国家的税法有特殊规定：

核心约束：发票已开具、双方已对账确认的金额，不能再从应付账款中扣减。

因此，必须通过现金收/付款项的方式单独处理扣款项。

1.2 业务场景

场景说明扣款方向
品质扣款货物质量问题，供应商赔款供应商→我方
迟延罚款供应商延期交付，被罚款供应商→我方
我方补偿我方责任，主动赔偿供应商我方→供应商
其他补偿价格调整、业务补偿等双方均可

二、功能范围

2.1 核心功能

功能编号功能名称功能说明优先级
F01现金扣款申请发起现金扣款单，关联原应付单据P0
F02扣款审核财务人员审核扣款事项P0
F03现金收/付款确认确认现金收付，更新账户余额P0
F04往来核销核销应付/应收，生成核销记录P0
F05凭证生成生成总账凭证，传递到凭证引擎P0
F06扣款查询按供应商/日期/状态查询扣款记录P1
F07统计报表按供应商/月份/原因统计扣款P2

2.2 功能边界

本模块负责：

  • 现金扣款的申请、审核、确认
  • 与应付管理的联动
  • 凭证模板的生成
  • 往来核销处理

本模块不负责：

  • 采购对账逻辑（由采购模块处理）
  • 银行转账处理（由现金管理模块处理）
  • 发票开具/作废（由发票模块处理）

3.2 流程详细说明

步骤1：采购对账发现扣款项

  • 采购人员与供应商对账
  • 发现品质问题/迟延等需要扣款的情况
  • 评估扣款金额是否需要走现金流程

步骤2：发起现金扣款申请

  • 采购人员进入系统，发起"现金扣款申请"
  • 填写信息：
  • 供应商（必填）
  • 扣款方向（供应商→我方 / 我方→供应商）
  • 扣款金额（必填）
  • 扣款原因（必填）
  • 关联单据（必填，原应付发票）
  • 上传扣款协议（如有）
  • 提交审核

步骤3：财务审核

  • 财务人员收到审核通知
  • 审核内容：
  • 扣款事由是否合理
  • 金额是否正确
  • 附件是否齐全
  • 通过/驳回

步骤4：生成现金扣款单

  • 审核通过后，系统自动生成"现金扣款单"
  • 分配唯一单据编号（格式：CASH-YYYYMMDD-XXX）
  • 状态更新为"待确认"

步骤5：现金收/付款确认

  • 财务人员执行收/付款操作
  • 选择支付方式：现金/银行转账
  • 确认收/付款信息
  • 提交确认

步骤6：往来核销

  • 系统自动进行往来核销
  • 更新供应商应付/应收余额
  • 生成核销记录

步骤7：生成凭证

  • 根据凭证模板生成凭证分录
  • 传递到总账凭证引擎
  • 完成整个流程

四、数据模型

4.1 核心表：cash_deduction


CREATE TABLE cash_deduction (
    -- 主键
    id              BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    deduction_no    VARCHAR(50) UNIQUE NOT NULL COMMENT '扣款单编号',
    
    -- 基本信息
    deduction_date  DATE NOT NULL COMMENT '扣款日期',
    supplier_id     BIGINT NOT NULL COMMENT '供应商ID',
    currency        VARCHAR(10) NOT NULL COMMENT '币种',
    
    -- 扣款信息
    direction       TINYINT NOT NULL COMMENT '扣款方向:1=供应商→我方,2=我方→供应商',
    amount          DECIMAL(18,4) NOT NULL COMMENT '扣款金额',
    reason          VARCHAR(50) NOT NULL COMMENT '扣款原因',
    reason_remark   VARCHAR(200) COMMENT '扣款原因备注',
    
    -- 关联信息
    source_type     VARCHAR(20) COMMENT '来源单据类型',
    source_id       BIGINT COMMENT '来源单据ID',
    original_amount DECIMAL(18,4) COMMENT '原应付金额',
    
    -- 核销信息
    writeoff_status TINYINT DEFAULT 0 COMMENT '核销状态:0=未核销,1=已核销',
    writeoff_time   DATETIME COMMENT '核销时间',
    
    -- 支付信息
    pay_method      VARCHAR(20) COMMENT '支付方式',
    pay_account     VARCHAR(50) COMMENT '收付款账号',
    pay_status      TINYINT DEFAULT 0 COMMENT '支付状态:0=未支付,1=已支付',
    pay_time        DATETIME COMMENT '支付时间',
    
    -- 凭证信息
    voucher_id      BIGINT COMMENT '凭证ID',
    
    -- 审核信息
    status          TINYINT DEFAULT 0 COMMENT '状态:0=草稿,1=审核中,2=待确认,3=已完成,4=已驳回',
    create_by       BIGINT NOT NULL COMMENT '创建人',
    create_time     DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_by       BIGINT COMMENT '更新人',
    update_time     DATETIME COMMENT '更新时间',
    audit_by        BIGINT COMMENT '审核人',
    audit_time      DATETIME COMMENT '审核时间',
    audit_remark    VARCHAR(200) COMMENT '审核意见',
    
    -- 附件
    attachment_ids  VARCHAR(500) COMMENT '附件ID列表(JSON)',
    
    -- 扩展字段
    ext_data        TEXT COMMENT '扩展数据(JSON)',
    
    -- 逻辑删除
    deleted         TINYINT DEFAULT 0 COMMENT '删除标记'
) COMMENT '现金扣款单表';

4.2 核销记录表：cash_deduction_writeoff


CREATE TABLE cash_deduction_writeoff (
    id                  BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    deduction_id        BIGINT NOT NULL COMMENT '扣款单ID',
    
    -- 核销信息
    writeoff_type       TINYINT NOT NULL COMMENT '核销类型:1=收款核销,2=付款核销',
    writeoff_amount     DECIMAL(18,4) NOT NULL COMMENT '核销金额',
    
    -- 关联信息
    ap_id               BIGINT COMMENT '应付单ID',
    ar_id               BIGINT COMMENT '应收单ID',
    
    -- 凭证信息
    voucher_id          BIGINT COMMENT '凭证ID',
    voucher_no          VARCHAR(50) COMMENT '凭证号',
    
    -- 审计字段
    create_by           BIGINT NOT NULL COMMENT '创建人',
    create_time         DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    
    INDEX idx_deduction (deduction_id),
    INDEX idx_ap (ap_id),
    INDEX idx_voucher (voucher_id)
) COMMENT '现金扣款核销记录表';

五、凭证模板

5.1 模板A：供应商→我方（我们收钱）

项目内容
模板代码CASH_DEDUCT_A
模板名称供应商赔款收款
业务描述供应商因品质问题/迟延等赔款给我们
借方科目1001（库存现金）/ 1002（银行存款）
贷方科目2202（应付账款-XX供应商）
辅助核算供应商、部门、项目

示例分录：


借：银行存款   1,000.00
贷：应付账款-XX供应商  1,000.00

摘要：收到XX供应商品质扣款补偿
辅助：供应商=XX公司，部门=采购部

5.2 模板B：我方→供应商（我们付钱）

项目内容
模板代码CASH_DEDUCT_B
模板名称我方赔款付款
业务描述我们因自身原因赔款给供应商
借方科目2202（应付账款-XX供应商）
贷方科目1001（库存现金）/ 1002（银行存款）
辅助核算供应商、部门、项目

示例分录：


借：应付账款-XX供应商  500.00
贷：银行存款    500.00

摘要：支付XX供应商迟延补偿款
辅助：供应商=XX公司，部门=生产部

字段校验规则
扣款金额> 0；≤ 原应付金额（如果设置了上限）
供应商必须是有效供应商
扣款日期≤ 当前日期
扣款方向必选
扣款原因必选；选择"其他"时，备注必填
来源单据必须是该供应商的有效应付单据

// 创建现金扣款单
POST /api/cash-deduction
Request: {
  deduction_date: "2026-04-17",
  supplier_id: 1001,
  currency: "VND",
  direction: 1,  // 1=供应商→我方
  amount: 1000000,
  reason: "QUALITY",
  reason_remark: "货物质量问题扣款",
  source_type: "AP_INVOICE",
  source_id: 2001,
  pay_method: "BANK",
  pay_account: "CNY-001"
}
Response: {
  code: 0,
  data: {
    deduction_id: 1001,
    deduction_no: "CASH-20260417-001"
  }
}

// 审核扣款单

十、非功能性需求

10.1 性能要求

指标要求
单据查询响应时间≤ 500ms
单据提交响应时间≤ 1s
批量导出支持导出1000条记录

10.2 安全要求

  • 所有操作记录操作日志
  • 敏感操作（如删除）需二次确认
  • 审核记录永久保存，不可删除

10.3 各国差异适配

国家特殊要求实现方式
越南必须通过现金处理必填扣款协议附件
泰国需双方书面确认支持附件上传
菲律宾无特殊要求正常流程
沙特VAT合规与VAT模块联动

十一、待确认事项

  1. 支付方式：是否支持多种支付方式（现金/转账/微信/支付宝）？
  2. 汇率处理：越南盾等小币种，是否需要汇率折算？
  3. 审批流：是否需要多级审批？
  4. 金额上限：单笔扣款是否需要设置上限？
  5. 历史数据：是否需要支持导入历史扣款数据？
