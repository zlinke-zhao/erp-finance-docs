## 一、模块概述

### 1.1 功能定位

银行日记账与对账管理是ERP财务系统的核心模块，负责：
- 维护企业所有银行账户的日记账记录
- 采集银行流水（手工/Excel/银企直连）
- 自动勾对银行流水与企业日记账
- 生成余额调节表，确保账账相符

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **渠道无关** | 对账引擎不关心银行流水来源，统一数据模型 |
| **可扩展** | 银企直连采用插件化设计，未来可扩展各银行API |
| **可追溯** | 银行流水 ↔ 企业日记账 ↔ 收款/付款单 ↔ 总账凭证 全链路关联 |

```
│  手工录入 / Excel导入 / 银企直连(预留)                           │

│  【对账处理】                                                   │
│  自动勾对 → 人工确认 → 余额调节                                  │
│        ↓                                                        │
│  【月末结账】                                                   │
│  余额调节表 → 存档 → 结账锁定                                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 银行流水采集流程

```
1. 网银导出Excel/CSV
       ↓
2. 下载ERP标准模板（或使用银行专属模板）
       ↓
3. 上传文件 → 系统自动解析
       ↓
4. 预览解析结果 → 修正错误行
       ↓
5. 确认入库
```

### 3.3 自动勾对流程

```
┌─────────────────────────────────────────┐
│           对账引擎                       │
├─────────────────────────────────────────┤
│                                         │
│  P1: 精准匹配（流水号=单据号）            │
│      → 直接勾对，无需确认                │
│                                         │
│  P2: 金额+日期匹配（±3天）               │
│      → 系统推荐，待确认                  │
│                                         │
│  P3: 对手+金额匹配                       │
│      → 系统推荐，待确认                  │
│                                         │
│  P4: 模糊推荐（金额误差<1%）              │
│      → 仅提示，不自动勾对                │
│                                         │
└─────────────────────────────────────────┘
```

## 四、数据模型

### 4.1 银行账户表 bank_account

```sql
CREATE TABLE bank_account (
    id                BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',

    -- === 基本信息 ===
    account_no        VARCHAR(32)  NOT NULL UNIQUE COMMENT '银行账号',
    bank_code         VARCHAR(12)  NOT NULL        COMMENT '银行代码(如ICBC/ABC)',
    bank_name         VARCHAR(100) NOT NULL        COMMENT '开户行全称',
    branch_name       VARCHAR(200) DEFAULT NULL     COMMENT '支行名称',
    account_name      VARCHAR(200) NOT NULL        COMMENT '账户名称(与印鉴一致)',
    account_type      VARCHAR(16)  NOT NULL        COMMENT '账户类型: BASIC(基本户)/GENERAL(一般户)/SPECIAL(专用)/TEMPORARY(临时)',

    -- === 财务信息 ===
    currency          VARCHAR(6)   NOT NULL DEFAULT 'CNY' COMMENT '币种',
    balance           DECIMAL(16,2) DEFAULT 0       COMMENT '当前余额',
    balance_date      DATE         DEFAULT NULL     COMMENT '余额截止日期',

    -- === 辅助信息 ===
    opening_date      DATE         DEFAULT NULL     COMMENT '开户日期',
    remark            VARCHAR(500) DEFAULT NULL     COMMENT '备注',

    -- === 状态 ===
    status            VARCHAR(16)  NOT NULL DEFAULT 'ACTIVE'
        COMMENT 'ACTIVE(启用)/FROZEN(冻结)/CLOSED(已销户)',

    -- === 审计字段 ===
    created_by        BIGINT       DEFAULT NULL     COMMENT '创建人ID',
    created_at        DATETIME     DEFAULT CURRENT_TIMESTAMP,
    updated_by        BIGINT       DEFAULT NULL     COMMENT '更新人ID',
    updated_at        DATETIME     DEFAULT NULL ON UPDATE CURRENT_TIMESTAMP,

    -- === 索引 ===
    INDEX idx_account_no (account_no),
    INDEX idx_bank_code (bank_code),
    INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='银行账户表';
```

## 六、自动勾对规则

### 6.1 匹配规则详解

| 优先级 | 规则名称 | 匹配条件 | 处理方式 |
|--------|----------|----------|----------|
| P1 | 精准匹配 | 银行流水号 = 业务单据号 | 自动勾对，无需确认 |
| P2 | 金额日期匹配 | 金额完全相等 + 日期在±3天内 | 系统推荐，待人工确认 |
| P3 | 对手金额匹配 | 对方名称包含关键词 + 金额相等 | 系统推荐，待人工确认 |
| P4 | 模糊推荐 | 金额误差<1% + 同期交易 | 仅提示，不自动勾对 |

### 6.2 匹配算法伪代码

```python
def auto_match(statement, diaries):
    """
    银行流水自动匹配算法
    """
    # P1: 精准匹配
    exact = find_by_serial_no(statement.serial_no)
    if exact and exact.amount == statement.amount:
        return MatchResult(P1, auto_confirm=True)

    # P2: 金额+日期匹配
    candidates = find_by_amount_and_date(
        statement.amount,
        statement.tx_date,
        days_range=3
    )
    if len(candidates) == 1:
        return MatchResult(P2, candidates[0], auto_confirm=False)
    elif len(candidates) > 1:
        return MatchResult(P2, candidates, requires_manual=True)

    # P3: 对手+金额匹配
    candidates = find_by_counterparty_and_amount(
        statement.counterparty_name,
        statement.amount
    )
    if len(candidates) == 1:
        return MatchResult(P3, candidates[0], auto_confirm=False)

    # P4: 模糊推荐
    candidates = find_fuzzy_matches(statement, diaries)
    return MatchResult(P4, candidates, recommend_only=True)
```

## 七、凭证生成规则

### 7.1 日记账登记时自动生成凭证

当收款单/付款单审核通过时，系统自动生成日记账并关联凭证：

| 场景 | 借方 | 贷方 | 摘要 |
|------|------|------|------|
| 收款核销 | 银行存款 | 应收账款 | 收XXX公司货款 |
| 付款核销 | 应付账款 | 银行存款 | 付XXX公司货款 |
| 预收款项 | 银行存款 | 预收账款 | 预收XXX公司款项 |

### 7.2 对账确认后的额外校验

对账勾对完成后，系统更新凭证状态为"**已银行核验**"，用于：
- 标记该笔业务已与银行流水核对无误
- 作为内部审计的参考依据

## 八、界面原型要点

### 8.1 银行账户列表

- 显示所有账户：账号、开户行、余额、状态
- 支持按状态筛选：全部/启用/停用

### 8.3 余额调节表

- 标准格式：银行余额 + 加减项 = 调节后余额
- 支持逐笔列明未达账项详情
- 支持导出PDF存档
