⚖️ 应收票据 vs 应付票据对照

同一票据管理模块在销售端（应收）和采购端（应付）的镜像关系对照
维度 应收票据（销售端） 应付票据（采购端）
业务角色 我方是收款人/持票人 我方是付款人/出票人
资金方向 钱会流进来 钱会流出去
初始动作 收到客户票据 → 登记入池 向供应商开票 → 票据开立
持有目的 等待到期兑付 / 提前贴现变现 争取付款时间缓冲 / 延迟现金流出
核心操作 背书转出 / 贴现 / 到期托收 到期付款 / 票据展期 / 接受背书抵付
会计科目 1121 应收票据（资产类） 2201 应付票据（负债类）
到期风险 承兑人拒付风险（信用风险） 到期资金不足风险（流动性风险）
预警关注 到期提醒 → 及时托收 到期提醒 → 备付金到位
凭证方向 收取: 借应收票据 / 贷应收账款 开立: 借应付账款 / 贷应付票据
背书关系 我方背书给下游供应商 上游供应商背书给我方（收票抵付）
与付款模块关系 替代实收 — 票据方式收款 替代实付 — 票据方式付款
